<task type="auto" tdd="true">
  <name>Task 2: Create DatabaseService with WAL mode, schema, integrity check, and corruption recovery</name>
  <files>JoJot/Services/DatabaseService.cs</files>
  <behavior>
    - OpenAsync creates directory and database file at specified path
    - After OpenAsync, PRAGMA journal_mode returns "wal"
    - After OpenAsync, PRAGMA synchronous returns 1 (NORMAL)
    - After EnsureSchemaAsync, all four tables exist in sqlite_master: notes, app_state, pending_moves, preferences
    - notes table has columns: id, desktop_guid, name, content, pinned, created_at, updated_at, sort_order, editor_scroll_offset, cursor_position
    - app_state table has columns: id, desktop_guid, desktop_name, desktop_index, window_left, window_top, window_width, window_height, active_tab_id, scroll_offset
    - pending_moves table has columns: id, window_id, from_desktop, to_desktop, detected_at
    - preferences table has columns: key, value
    - EnsureSchemaAsync is idempotent (calling twice does not error)
    - VerifyIntegrityAsync returns true when all four tables exist
    - VerifyIntegrityAsync returns false when a table is missing
    - ExecuteNonQueryAsync serializes concurrent writes (only one executes at a time)
    - HandleCorruptionAsync renames corrupt file to .corrupt and recreates a fresh database with schema
    - RunPendingMigrationsAsync exists as a stub (no migrations in Phase 1; logs "no pending migrations")
  </behavior>
  <action>
Create JoJot/Services/DatabaseService.cs:
- Public static class `DatabaseService` in namespace `JoJot.Services`
- Private static fields: `SqliteConnection? _connection`, `SemaphoreSlim _writeLock = new(1, 1)`, `string _dbPath = string.Empty`
- `OpenAsync(string dbPath)`:
  - Store dbPath in _dbPath field
  - `Directory.CreateDirectory(Path.GetDirectoryName(dbPath)!)`
  - Build connection string with `SqliteConnectionStringBuilder { DataSource = dbPath, Mode = SqliteOpenMode.ReadWriteCreate }` — do NOT set Cache=Shared
  - Open connection, then execute PRAGMAs: `PRAGMA journal_mode=WAL;`, `PRAGMA synchronous=NORMAL;`, `PRAGMA foreign_keys=ON;`
- `EnsureSchemaAsync()`:
  - CREATE TABLE IF NOT EXISTS for all four tables with exact column definitions per DATA-03, DATA-04, DATA-05, DATA-06
  - notes: id INTEGER PRIMARY KEY AUTOINCREMENT, desktop_guid TEXT NOT NULL, name TEXT, content TEXT NOT NULL DEFAULT '', pinned INTEGER NOT NULL DEFAULT 0, created_at TEXT NOT NULL DEFAULT (datetime('now')), updated_at TEXT NOT NULL DEFAULT (datetime('now')), sort_order INTEGER NOT NULL DEFAULT 0, editor_scroll_offset INTEGER NOT NULL DEFAULT 0, cursor_position INTEGER NOT NULL DEFAULT 0
  - app_state: id INTEGER PRIMARY KEY AUTOINCREMENT, desktop_guid TEXT NOT NULL UNIQUE, desktop_name TEXT, desktop_index INTEGER, window_left REAL, window_top REAL, window_width REAL, window_height REAL, active_tab_id INTEGER, scroll_offset REAL
  - pending_moves: id INTEGER PRIMARY KEY AUTOINCREMENT, window_id TEXT NOT NULL, from_desktop TEXT NOT NULL, to_desktop TEXT, detected_at TEXT NOT NULL DEFAULT (datetime('now'))
  - preferences: key TEXT PRIMARY KEY, value TEXT NOT NULL
- `VerifyIntegrityAsync()`: Query sqlite_master for each of the four table names; return true only if all exist. If any missing, log warning via LogService and run `PRAGMA quick_check`. Return false if quick_check fails or tables missing.
- `HandleCorruptionAsync(string dbPath)`: Dispose connection, rename dbPath to dbPath+".corrupt" (delete existing .corrupt first), then call OpenAsync + EnsureSchemaAsync to recreate.
- `ExecuteNonQueryAsync(string sql)`: Acquire _writeLock, create command, execute, release lock. Public for schema creation; also used internally.
- `ExecuteScalarAsync<T>(string sql)`: Same lock pattern, returns scalar.
- `ExecuteReaderAsync(string sql, Action<SqliteDataReader> handler)`: Read lock pattern for queries.
- `RunPendingMigrationsAsync()`: Stub — `LogService.Info("No pending migrations");` Returns Task.CompletedTask.
- `CloseAsync()`: Dispose connection, set to null.
- All database exceptions in non-schema methods should be caught, logged, and re-thrown (so callers can handle).
  </action>
