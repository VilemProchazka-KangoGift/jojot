## Common Pitfalls

### Pitfall 1: Inconsistent Traceability Status Values
**What goes wrong:** REQUIREMENTS.md uses mixed status values ("Pending", "Done", "Complete")
**Why it happens:** Different phases used different terminology
**How to avoid:** Standardize all to "Complete" across the board

### Pitfall 2: Stale Coverage Footer
**What goes wrong:** Coverage footer shows partial counts from older phases
**Why it happens:** Footer not updated when gap closure phases mark requirements complete
**How to avoid:** Recalculate all counts: total, mapped, satisfied

### Pitfall 3: Missing Phase 10.1 Context
**What goes wrong:** Verification report verifies pre-10.1 code state
**Why it happens:** Phase 10.1 applied integration fixes after Phase 9/10 execution
**How to avoid:** Verify against current codebase (post-10.1 state)
