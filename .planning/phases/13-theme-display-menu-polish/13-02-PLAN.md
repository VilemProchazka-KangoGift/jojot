  <name>Task 1: Fix hamburger menu dismiss after submenu interaction (WIN-02)</name>
  <files>JoJot/MainWindow.xaml.cs</files>
  <action>
The bug: When the user hovers over "Delete older than" (opening the submenu), `HamburgerMenu.StaysOpen` is set to `true`. If the user then clicks outside (not on a submenu item, not back on the menu), the `ScheduleSubmenuClose` timer may fire after the click event, or the click may land in a gap where neither the submenu border's MouseLeave nor the menu item's MouseLeave has fired. The `StaysOpen=true` prevents WPF's built-in light-dismiss from closing the popup.

The fix has two parts:

**Part 1 — Add a Closed handler for DeleteOlderSubmenu:**

In the constructor (after the existing `HamburgerMenu.Closed` handler around line 140), add:
```csharp
// Phase 13: Ensure StaysOpen resets whenever submenu closes (WIN-02)
DeleteOlderSubmenu.Closed += (s, e) =>
{
    HamburgerMenu.StaysOpen = false;
};
```
This is the safety net: no matter HOW the submenu closes (timer, click outside, programmatic), `StaysOpen` gets reset so the main popup regains light-dismiss behavior.

**Part 2 — Add a PreviewMouseDown handler on the main window to force-close:**

Register in the constructor (near the other event handlers, after line 147):
```csharp
// Phase 13: Force-close hamburger menu on any click outside (WIN-02)
PreviewMouseDown += (s, e) =>
{
    if (HamburgerMenu.IsOpen
        && !IsMouseOverPopup(HamburgerMenu)
        && !IsMouseOverPopup(DeleteOlderSubmenu)
        && !IsMouseOverElement(HamburgerButton))
    {
        DeleteOlderSubmenu.IsOpen = false;
        HamburgerMenu.IsOpen = false;
    }
};
```

Add helper methods to the class (near the other hamburger-related methods around line 2047):
```csharp
/// <summary>
/// Checks if the mouse is currently over any visual within a Popup.
/// Used for hamburger menu dismiss detection (WIN-02).
/// </summary>
private static bool IsMouseOverPopup(Popup popup)
{
    return popup.Child is FrameworkElement child && child.IsMouseOver;
}

private static bool IsMouseOverElement(UIElement element)
{
    return element.IsMouseOver;
}
```

This ensures:
- Clicking inside the menu or submenu does NOT dismiss (correct behavior)
- Clicking on the hamburger button itself does NOT dismiss (toggle handled by HamburgerButton_Click)
- Clicking anywhere else forces both popups closed immediately
- The StaysOpen flag is always cleaned up via the Closed event handlers
