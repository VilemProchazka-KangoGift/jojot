  <files>JoJot/ViewModels/MainWindowViewModel.cs, JoJot.Tests/ViewModels/FindEngineTests.cs</files>
  <behavior>
    - FindAllMatches(content, query, caseSensitive: false, wholeWord: false) — default params preserve backward compat
    - Case-sensitive: "Hello" matches "Hello" but not "hello" when caseSensitive=true
    - Whole-word: "the" matches "the" but not "them" or "bathe" when wholeWord=true (word boundary = non-alphanumeric or string edge)
    - Both options combined: caseSensitive=true + wholeWord=true
    - ReplaceAll(content, query, replacement, caseSensitive, wholeWord) returns (newContent, replacementCount)
    - ReplaceAll preserves text outside matches, handles empty replacement, handles overlapping boundaries
    - ReplaceAll with no matches returns (originalContent, 0)
    - ReplaceSingle(content, matchIndex, queryLength, replacement) returns new content with single match replaced
  </behavior>
  <action>
    1. Add tests first to FindEngineTests.cs for:
       - FindAllMatches with caseSensitive=true (exact case matching)
       - FindAllMatches with wholeWord=true (word boundary detection: check start/end of string, spaces, punctuation, mixed alphanumeric boundaries)
       - FindAllMatches with both caseSensitive=true and wholeWord=true
       - ReplaceAll: basic replacement, case-sensitive replacement, whole-word replacement, empty replacement (deletion), no matches returns 0 count
       - ReplaceSingle: replaces at specific match position, returns updated content

    2. Run tests — they should fail (RED).

    3. Extend MainWindowViewModel:
       - Change `FindAllMatches` signature to: `internal static List<int> FindAllMatches(string content, string query, bool caseSensitive = false, bool wholeWord = false)`
         - When caseSensitive=false use StringComparison.OrdinalIgnoreCase (existing behavior)
         - When caseSensitive=true use StringComparison.Ordinal
         - When wholeWord=true, after finding a match at index, check that char before match (if any) is not a letter/digit and char after match end (if any) is not a letter/digit. Skip match if boundary check fails.
       - Add `internal static (string NewContent, int Count) ReplaceAll(string content, string query, string replacement, bool caseSensitive = false, bool wholeWord = false)`
         - Use FindAllMatches to get positions, then build new string by replacing from end to start (to preserve indices) or use StringBuilder forward pass.
         - Return tuple of new content and replacement count.
       - Add `internal static string ReplaceSingle(string content, int matchIndex, int queryLength, string replacement)`
         - Simple string surgery: content[..matchIndex] + replacement + content[(matchIndex + queryLength)..]

    4. Run tests — they should pass (GREEN).

    Note: Existing FindAllMatches callers use the 2-param overload which still works via default params. No breaking changes.
  </action>
