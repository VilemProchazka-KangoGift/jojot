**MainWindow.xaml changes:**

1. **Update Grid.ColumnDefinitions** (lines 59-63):
   ```xml
   <Grid.ColumnDefinitions>
       <ColumnDefinition x:Name="TabPanelColumn" Width="180" MinWidth="120" MaxWidth="400"/>
       <ColumnDefinition Width="Auto"/>
       <ColumnDefinition Width="*"/>
   </Grid.ColumnDefinitions>
   ```
   - Column 0: Give it `x:Name="TabPanelColumn"` for code-behind access, add `MinWidth="120"` and `MaxWidth="400"`
   - Column 1: Change from `Width="1"` to `Width="Auto"` (GridSplitter determines its own width)
   - Column 2: Keep as `*`

2. **Replace the static Border separator** (line 130):
   Replace:
   ```xml
   <Border Grid.Column="1" Background="{DynamicResource c-border}"/>
   ```
   With:
   ```xml
   <GridSplitter Grid.Column="1" Width="4" HorizontalAlignment="Center"
                  VerticalAlignment="Stretch" Background="{DynamicResource c-border}"
                  Cursor="SizeWE" ResizeBehavior="PreviousAndNext"
                  DragCompleted="TabPanelSplitter_DragCompleted"/>
   ```
   - Width 4px gives a comfortable drag target (wider than the old 1px border but still subtle)
   - `ResizeBehavior="PreviousAndNext"` resizes Column 0 and Column 2
   - `DragCompleted` event fires once after the user finishes dragging — saves to preferences

**MainWindow.xaml.cs changes:**

1. **Load persisted panel width at startup:**
   Find the constructor or the method that runs after InitializeComponent (likely `LoadSessionAsync` or equivalent startup path). Add panel width restoration:
   ```csharp
   private async Task RestoreTabPanelWidthAsync()
   {
       var saved = await DatabaseService.GetPreferenceAsync("tab_panel_width");
       if (saved != null && double.TryParse(saved, System.Globalization.CultureInfo.InvariantCulture, out double width))
       {
           // Clamp to min/max defined in XAML
           width = Math.Clamp(width, 120, 400);
           TabPanelColumn.Width = new GridLength(width);
       }
   }
   ```
   Call `await RestoreTabPanelWidthAsync()` early in the window initialization, after `InitializeComponent()` but before tab loading. A good place is at the end of the constructor or at the start of the session loading method. Look for where the window first sets up — likely in the constructor after `InitializeComponent()` or in an async init method called from the constructor.

2. **Save panel width on drag complete:**
   ```csharp
   private async void TabPanelSplitter_DragCompleted(object sender, System.Windows.Controls.Primitives.DragCompletedEventArgs e)
   {
       var width = TabPanelColumn.ActualWidth;
       await DatabaseService.SetPreferenceAsync("tab_panel_width",
           width.ToString(System.Globalization.CultureInfo.InvariantCulture));
   }
   ```

**Important notes:**
- The `x:Name="TabPanelColumn"` on the ColumnDefinition makes it accessible as a field in the code-behind automatically (WPF named element).
- Use `CultureInfo.InvariantCulture` for parsing/formatting to avoid locale-dependent decimal separators.
- The MinWidth/MaxWidth on the ColumnDefinition enforces constraints — the GridSplitter respects these automatically.
