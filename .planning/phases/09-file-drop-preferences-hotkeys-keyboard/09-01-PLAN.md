<task type="auto">
  <name>Task 1: Create FileDropService with content inspection</name>
  <files>JoJot/Services/FileDropService.cs</files>
  <action>
Create a static FileDropService class in JoJot/Services/ following the existing static service pattern (like ThemeService, AutosaveService).

The service provides:

1. **IsBinaryContent(byte[] buffer, int bytesRead) -> bool**: Scans buffer for binary indicators:
   - Null byte (0x00) = definitive binary
   - Non-printable chars below 0x08
   - Chars between 0x0E and 0x1F (excluding 0x1B/ESC) = binary
   - Tab (0x09), LF (0x0A), CR (0x0D) are allowed (common text whitespace)

2. **ValidateFileAsync(string filePath) -> FileDropResult**: Checks a single file:
   - Check FileInfo.Length > 500 * 1024 -> return error "too large (max 500KB)"
   - Read first 8192 bytes (or full file if smaller) into buffer
   - Call IsBinaryContent -> return error "contains binary content"
   - If valid: read full content via File.ReadAllTextAsync(path)
   - Return FileDropResult with either Content + FileName or ErrorMessage

3. **ProcessDroppedFilesAsync(string[] filePaths) -> FileDropSummary**: Processes multiple files:
   - Iterate each path, call ValidateFileAsync
   - Collect valid results (Content + FileName) and error results
   - Return summary with ValidFiles list, ErrorCount, and a combined error message string

Define result types:
```csharp
public record FileDropResult(bool IsValid, string FileName, string? Content, string? ErrorMessage);
public record FileDropSummary(List<FileDropResult> ValidFiles, int ErrorCount, string? CombinedErrorMessage);
```

For CombinedErrorMessage format:
- If validCount > 0 and errorCount > 0: "{validCount} file(s) opened, {errorCount} skipped ({lastError})"
- If validCount == 0: the last error message
- If errorCount == 0: null

Use System.IO for file operations. Wrap file reads in try/catch — on IOException or UnauthorizedAccessException, return error "Failed to read '{filename}'". Log errors via LogService.Error.

**XAML changes in MainWindow.xaml:**

1. Add `AllowDrop="True"` to the main content Grid (the one containing the editor area — the Grid that holds the content column, NOT the root Grid). Add event handlers: `DragEnter="OnFileDragEnter"`, `DragLeave="OnFileDragLeave"`, `DragOver="OnFileDragOver"`, `Drop="OnFileDrop"`.

2. Add a drop overlay Grid (named `FileDropOverlay`) as the LAST child of the content area Grid (so it renders on top). Visibility="Collapsed" by default. Structure:
   ```xml
   <Grid x:Name="FileDropOverlay" Visibility="Collapsed" Background="#80000000" ZIndex="100" IsHitTestVisible="True"
         AllowDrop="True" DragEnter="OnFileDragEnter" DragOver="OnFileDragOver" Drop="OnFileDrop">
       <Border BorderBrush="{DynamicResource c-accent}" BorderThickness="3" CornerRadius="8" Margin="16">
           <StackPanel HorizontalAlignment="Center" VerticalAlignment="Center">
               <TextBlock Text="&#xE896;" FontFamily="Segoe Fluent Icons" FontSize="48"
                          Foreground="{DynamicResource c-accent}" HorizontalAlignment="Center"/>
               <TextBlock Text="Drop file here" FontSize="18" FontWeight="SemiBold"
                          Foreground="White" HorizontalAlignment="Center" Margin="0,12,0,0"/>
               <TextBlock Text="Text files only, max 500KB" FontSize="12"
                          Foreground="#AAAAAA" HorizontalAlignment="Center" Margin="0,4,0,0"/>
           </StackPanel>
       </Border>
   </Grid>
   ```

**Code-behind changes in MainWindow.xaml.cs:**

1. Add the four drag event handlers:

   **OnFileDragEnter**: Check `e.Data.GetDataPresent(DataFormats.FileDrop)`. If true: `e.Effects = DragDropEffects.Copy`, show `FileDropOverlay` (Visibility.Visible). If false: `e.Effects = DragDropEffects.None`. Set `e.Handled = true`.

   **OnFileDragOver**: Same as DragEnter (set Effects + Handled). This prevents the "no drop" cursor from appearing.

   **OnFileDragLeave**: Use a delayed hide pattern to avoid flicker when entering child elements. Check if the mouse position (from `e.GetPosition(this)`) is outside the window bounds. If outside: hide `FileDropOverlay` (Visibility.Collapsed). Set `e.Handled = true`. Alternatively, since the overlay itself has AllowDrop and covers everything, handle DragLeave ONLY on the overlay — when mouse truly leaves the overlay (and thus the window), hide it.

   **OnFileDrop**: Hide `FileDropOverlay`. Extract `string[] files = (string[])e.Data.GetData(DataFormats.FileDrop)`. Call `_ = ProcessDroppedFilesAsync(files)`. Set `e.Handled = true`.

2. Add async method `ProcessDroppedFilesAsync(string[] filePaths)`:
   - Call `FileDropService.ProcessDroppedFilesAsync(filePaths)` to get the summary
   - For each valid file in summary.ValidFiles:
     - Get max sort_order from current tabs (or 0 if empty)
     - Call `DatabaseService.CreateNoteAsync(_desktopGuid, result.FileName, result.Content, sortOrder++)`
     - Create NoteTab in-memory, add to _tabs ObservableCollection, build ListBoxItem, add to TabList
   - Select the LAST valid tab as active (per CONTEXT.md: last valid file becomes active)
   - If summary.ErrorCount > 0: show error toast using existing ShowToast pattern but with just the message (no undo button). Pass the CombinedErrorMessage. The toast auto-dismisses after 4 seconds per existing behavior.

IMPORTANT: Follow existing tab creation patterns from CreateNewTabAsync. The new tabs should appear at the bottom of the unpinned section (after existing tabs). Use BuildTabItem or the equivalent existing method to create the ListBoxItem for each new tab.

IMPORTANT: Do NOT interfere with the existing tab drag-reorder (which uses raw mouse events, not DragDrop API). The AllowDrop for file drops should be on the content/editor area, not on the tab list. If both need AllowDrop, ensure the tab list's existing reorder logic is preserved.
