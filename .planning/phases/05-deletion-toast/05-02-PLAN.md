**1. Add Ctrl+W handler to `Window_PreviewKeyDown`:**

Insert a new key handler block in the existing `Window_PreviewKeyDown` method (before the existing Ctrl+T, Ctrl+F etc. handlers or in logical order). The handler should:

```csharp
// Ctrl+W: Delete active tab (TDEL-01)
if (e.Key == Key.W && Keyboard.Modifiers == ModifierKeys.Control)
{
    if (_activeTab != null)
    {
        _ = DeleteTabAsync(_activeTab);
        e.Handled = true;
    }
    return;
}
```

Place this BEFORE any handler that might consume Key.W. Use `PreviewKeyDown` (tunneling) which fires before focused control's KeyDown — this is already the pattern used for Ctrl+T, Ctrl+K, Ctrl+P.

Set `e.Handled = true` to prevent any further processing of Ctrl+W (e.g., ContentEditor consuming it).

**2. Add middle-click handler to each tab's ListBoxItem in `CreateTabListItem`:**

In the `CreateTabListItem` method, add a `PreviewMouseButtonDown` handler to the outer `ListBoxItem` (the `item` variable). Must use `Preview` (tunneling) to intercept BEFORE WPF selection logic:

```csharp
item.PreviewMouseButtonDown += (s, e) =>
{
    if (e.ChangedButton == MouseButton.Middle)
    {
        _ = DeleteTabAsync(tab);
        e.Handled = true; // Prevent WPF auto-scroll on middle-click
    }
};
```

The lambda captures the `tab` parameter directly (each CreateTabListItem call gets its own `tab` binding). Do NOT use `ListBoxItem.Tag` — use the captured `tab` reference.

**Important:** Set `e.Handled = true` to prevent WPF from initiating auto-scroll behavior on middle-click. Without this, the ListBox may start a scroll-drag operation.

**Important:** The middle-click handler must NOT interfere with the existing drag-to-reorder logic (which uses left-button `PreviewMouseLeftButtonDown`). Since we check `e.ChangedButton == MouseButton.Middle`, there is no conflict — left-button events are unaffected.
  </action>

private static void AnimateOpacity(UIElement element, double from, double to, int durationMs)

    var anim = new System.Windows.Media.Animation.DoubleAnimation(from, to, TimeSpan.FromMilliseconds(durationMs));
    element.BeginAnimation(UIElement.OpacityProperty, anim);
}

**2. Add x delete icon to each tab in `CreateTabListItem`:**

Inside `CreateTabListItem`, after the inner `grid` is created and populated (with the label row and date row), add a `TextBlock` overlay for the x icon:

```csharp
var deleteIcon = new TextBlock
{
    Text = "\u00D7",  // × multiplication sign (clean cross shape)
    FontSize = 12,
    Foreground = MutedTextBrush,   // #888888 per locked decision
    Opacity = 0,                    // Hidden until hover
    HorizontalAlignment = HorizontalAlignment.Right,
    VerticalAlignment = VerticalAlignment.Top,
    Margin = new Thickness(0, 2, 4, 0),
    Cursor = Cursors.Hand,
    IsHitTestVisible = true
};
Grid.SetRowSpan(deleteIcon, 2);  // Span both rows of the grid
grid.Children.Add(deleteIcon);   // Add LAST so it renders on top
```

**3. Wire hover show/hide on the outer border's MouseEnter/MouseLeave:**

The `outerBorder` already has `MouseEnter` and `MouseLeave` handlers for the hover background effect. EXTEND these handlers (do not replace them) to also animate the delete icon:

In the `MouseEnter` handler, add:
```csharp
AnimateOpacity(deleteIcon, 0, 1, 100);  // 100ms fade-in per TDEL-03
```

In the `MouseLeave` handler, add:
```csharp
AnimateOpacity(deleteIcon, 1, 0, 100);  // Fade out
```

WPF `MouseLeave` on the outer border only fires when the mouse exits the entire subtree (including children like deleteIcon), so there is no flicker when moving from the tab area to the x icon. Verify this during implementation — if flicker occurs, use `IsMouseOver` check in MouseLeave before hiding.

**4. Wire color change on x icon hover:**

```csharp
deleteIcon.MouseEnter += (s, e) =>
    deleteIcon.Foreground = new System.Windows.Media.SolidColorBrush(
        System.Windows.Media.Color.FromRgb(0xe7, 0x4c, 0x3c));  // Red on hover

deleteIcon.MouseLeave += (s, e) =>
    deleteIcon.Foreground = MutedTextBrush;  // Back to gray
```

**5. Wire click to delete:**

```csharp
deleteIcon.MouseLeftButtonDown += (s, e) =>
{
    _ = DeleteTabAsync(tab);
    e.Handled = true;  // Prevent event from bubbling to ListBoxItem selection
};
```

**Important:** The `e.Handled = true` on the x icon click prevents the click from propagating to the ListBoxItem, which would trigger tab selection. The user expects clicking x to delete, not to select-then-delete.

**Important:** The deleteIcon must be added to the `grid` (the inner Grid), NOT to the `outerBorder`. It should be a child of the same Grid that contains the label and date TextBlocks, using `Grid.SetRowSpan(deleteIcon, 2)` to span both rows.

**6. Ensure delete icon is hidden during drag and rename:**

No special handling needed — the drag state uses `_isDragging` flag and captures mouse on the item. The x icon click handler fires on `MouseLeftButtonDown` which is distinct from the drag threshold (5px movement). If a drag is in progress (`_isDragging == true`), the mouseDown won't fire because mouse is captured by the drag item. No conflict expected.

For rename: the rename TextBox overlays the label area, not the x icon area. No conflict.
