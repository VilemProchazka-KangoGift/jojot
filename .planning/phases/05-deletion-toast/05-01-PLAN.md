**Add the following methods to MainWindow.xaml.cs:**

1. **`CommitPendingDeletionAsync()`** — If `_pendingDeletion` is not null: capture reference, set `_pendingDeletion = null`, cancel CTS, dispose CTS, then hard-delete all held tabs via `DatabaseService.DeleteNoteAsync(tab.Id)` for each tab. Null-out `_pendingDeletion` BEFORE disposing CTS to prevent double-dispose races.

2. **`DeleteTabAsync(NoteTab tab)`** — The single-tab delete entry point:
   - Call `SaveCurrentTabContent()` first (save editor before any mutation)
   - Call `await CommitPendingDeletionAsync()` (commit any in-flight pending deletion)
   - Capture `int originalIndex = _tabs.IndexOf(tab)` before removal
   - Check if the deleted tab is the active tab: `bool wasActive = (_activeTab?.Id == tab.Id)`
   - Remove tab from `_tabs` collection: `_tabs.Remove(tab)`
   - Call `RebuildTabList()`
   - If `wasActive`: call `await ApplyFocusCascadeAsync(originalIndex)`
   - Create new `CancellationTokenSource`, store `_pendingDeletion = new PendingDeletion([tab], [originalIndex], cts)`
   - Call `ShowToast(isBulk: false, label: tab.DisplayLabel)`
   - Fire-and-forget: `_ = StartDismissTimerAsync(cts.Token)`

3. **`DeleteMultipleAsync(IEnumerable<NoteTab> candidates)`** — Bulk delete (TOST-05, TDEL-06):
   - Filter: `var toDelete = candidates.Where(t => !t.Pinned).ToList()` (TDEL-06: skip pinned)
   - If empty list, return
   - `SaveCurrentTabContent()`
   - `await CommitPendingDeletionAsync()`
   - Capture original indexes before any removal: `var originalIndexes = toDelete.Select(t => _tabs.IndexOf(t)).ToList()`
   - Check if active tab is in the delete set: `bool wasActive = _activeTab != null && toDelete.Any(t => t.Id == _activeTab.Id)`
   - Capture active tab's original index if wasActive
   - Remove all tabs from `_tabs`
   - `RebuildTabList()`
   - If wasActive: `await ApplyFocusCascadeAsync(activeOriginalIndex)`
   - Create CTS, store PendingDeletion with toDelete and originalIndexes
   - `ShowToast(isBulk: true, count: toDelete.Count)`
   - `_ = StartDismissTimerAsync(cts.Token)`

4. **`StartDismissTimerAsync(CancellationToken token)`** — Async timer:
   - `await Task.Delay(4000, token)` in try/catch
   - On completion (no cancel): `await CommitPendingDeletionAsync()` then `HideToast()`
   - On `OperationCanceledException`: do nothing (return silently)

5. **`UndoDeleteAsync()`** — Undo handler:
   - If `_pendingDeletion` is null, return
   - Capture reference, set `_pendingDeletion = null`
   - Cancel CTS, dispose CTS (cancels the timer — no commit will happen)
   - Re-insert tabs at original positions: sort pairs by OriginalIndex ascending, insert each at `Math.Min(originalIndex, _tabs.Count)` — inserting in ascending order with clamping handles shifted indexes correctly
   - Call `RebuildTabList()`
   - Select the first restored tab: `SelectTabByNote(pending.Tabs[0])`
   - Call `HideToast()`

6. **`ApplyFocusCascadeAsync(int deletedIndex)`** — Focus cascade per TDEL-05:
   - Get visible tabs: `var visible = _tabs.Where(t => MatchesSearch(t)).ToList()`
   - If `visible.Count > 0`:
     - The tab at `deletedIndex` in the full `_tabs` was removed, so the tab now at index `Math.Min(deletedIndex, _tabs.Count - 1)` is the "first tab below". But we must pick from the visible (search-filtered) list.
     - Find the first visible tab whose position in `_tabs` is >= deletedIndex. If none, take the last visible tab. Select it via `SelectTabByNote`.
   - Else if `_searchText` is not empty (search active, no visible tabs remain):
     - Clear search: set `SearchBox.Text = ""`, `_searchText = ""`, `RebuildTabList()`
     - Recurse once: `await ApplyFocusCascadeAsync(0)` (cascade on full list with index 0)
   - Else (no tabs at all):
     - `await CreateNewTabAsync()` (auto-create empty tab — no empty state)

**Important edge case:** When `FlushAndClose()` is called (window closing), commit any pending deletion immediately so data is not lost. Add `_ = CommitPendingDeletionAsync()` at the start of `FlushAndClose()` (fire-and-forget is OK since the DB write is fast and the process stays alive).
