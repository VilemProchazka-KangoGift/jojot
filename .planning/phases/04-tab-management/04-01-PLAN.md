  <files>JoJot/Models/NoteTab.cs</files>
  <action>
Create `JoJot/Models/NoteTab.cs` in the Models directory. This is a plain C# class (no INotifyPropertyChanged — project uses code-behind, not MVVM binding).

**Properties (map 1:1 to notes table):**
- `long Id` — primary key
- `string DesktopGuid` — init to ""
- `string? Name` — custom name, nullable (null = no custom name)
- `string Content` — init to ""
- `bool Pinned` — init to false
- `DateTime CreatedAt`
- `DateTime UpdatedAt`
- `int SortOrder`
- `int EditorScrollOffset`
- `int CursorPosition`

**Computed properties (TABS-02, TABS-03):**

`string DisplayLabel`:
- If `Name` is not null and not whitespace: return `Name`
- Else if `Content` is not null and not whitespace: return first ~30 chars of `Content.Trim()` (break at char boundary, no mid-word truncation needed — TextTrimming handles that in UI)
- Else: return `"New note"`

`bool IsPlaceholder`:
- True when `Name` is null AND `Content` is null-or-whitespace
- Used by UI to apply muted italic styling

`string CreatedDisplay` — calls `FormatRelativeDate(CreatedAt)`:
- Same date as now → `"h:mm tt"` (e.g., "2:30 PM")
- Yesterday → `"Yesterday"`
- Same year → `"MMM d"` (e.g., "Mar 1")
- Different year → `"MMM d, yyyy"` (e.g., "Jan 15, 2025")

`string UpdatedDisplay` — calls `FormatRelativeTime(UpdatedAt)`:
- Less than 1 minute ago → `"Just now"`
- Less than 60 minutes ago → `"{N} min ago"`
- Same date → `"Today h:mm tt"` (e.g., "Today 2:30 PM")
- Yesterday → `"Yesterday"`
- Same year → `"MMM d"`
- Different year → `"MMM d, yyyy"`

Add a `void RefreshDisplayProperties()` method that does nothing (placeholder — the UI will call this to signal a refresh is needed after Name or Content changes). This is NOT INotifyPropertyChanged; the tab panel code-behind will explicitly re-read DisplayLabel/CreatedDisplay/UpdatedDisplay when it needs to update the visual.

**Important:** Use `DateTime.Now` (local time) for all comparisons. The database stores `datetime('now')` which is UTC — the reader in DatabaseService will parse with `DateTime.Parse` which converts to local. Ensure the model works correctly with local DateTime values.

<task type="auto">
  <name>Task 2: Add notes CRUD methods to DatabaseService</name>
  <files>JoJot/Services/DatabaseService.cs</files>
  <action>
Add the following methods to `DatabaseService.cs`, after the existing Window Geometry section. Follow the exact same pattern as `GetWindowGeometryAsync`/`SaveWindowGeometryAsync`: acquire `_writeLock`, try/catch with `LogService.Error`, parameterized queries, release in finally.

**GetNotesForDesktopAsync(string desktopGuid) -> Task&lt;List&lt;NoteTab>>:**
```sql
SELECT id, desktop_guid, name, content, pinned, created_at, updated_at,
       sort_order, editor_scroll_offset, cursor_position
FROM notes WHERE desktop_guid = @guid
ORDER BY pinned DESC, sort_order ASC
```
Parse each row into a NoteTab. For `created_at` and `updated_at`, use `DateTime.Parse(reader.GetString(col))`. For `pinned`, convert `reader.GetInt32(col) != 0`.

**InsertNoteAsync(string desktopGuid, string? name, string content, bool pinned, int sortOrder) -> Task&lt;long>:**
```sql
INSERT INTO notes (desktop_guid, name, content, pinned, sort_order)
VALUES (@guid, @name, @content, @pinned, @sortOrder);
SELECT last_insert_rowid();
```
Use a single command with two statements. Execute with `ExecuteScalarAsync` internally (but implement inline with the lock pattern, not by calling the public method — same as existing CRUD). Return the new row ID as `long`. Set `@name` to `DBNull.Value` when null. Set `@pinned` to `1` or `0` (integer).

**UpdateNoteContentAsync(long noteId, string content) -> Task:**
```sql
UPDATE notes SET content = @content, updated_at = datetime('now') WHERE id = @id
```

**UpdateNoteNameAsync(long noteId, string? name) -> Task:**
```sql
UPDATE notes SET name = @name, updated_at = datetime('now') WHERE id = @id
```
Set `@name` to `DBNull.Value` when null.

**UpdateNotePinnedAsync(long noteId, bool pinned) -> Task:**
```sql
UPDATE notes SET pinned = @pinned, updated_at = datetime('now') WHERE id = @id
```

**UpdateNoteSortOrdersAsync(IEnumerable&lt;(long Id, int SortOrder)> updates) -> Task:**
Execute all updates within a single lock acquisition (one await _writeLock.WaitAsync at the top). Loop through the updates and execute each:
```sql
UPDATE notes SET sort_order = @sortOrder WHERE id = @id
```
Do NOT use a transaction — SQLite auto-commits each statement, and the write lock already serializes. Keep it simple.

**DeleteNoteAsync(long noteId) -> Task:**
```sql
DELETE FROM notes WHERE id = @id
```

**GetMaxSortOrderAsync(string desktopGuid) -> Task&lt;int>:**
```sql
SELECT COALESCE(MAX(sort_order), -1) FROM notes WHERE desktop_guid = @guid
```
Returns -1 if no notes exist, so callers can do `maxSort + 1` for new notes.
