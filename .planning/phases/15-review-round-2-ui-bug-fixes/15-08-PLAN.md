<task type="auto">
  <name>Task 2: Fix drag ghost visibility and suppress hover during drag</name>
  <files>JoJot/MainWindow.xaml.cs</files>
  <action>
Fix three bugs preventing the drag ghost from working and causing visual artifacts during drag.

**Bug 1: Mouse.Capture uses wrong CaptureMode (line ~1392)**

Change:
```csharp
Mouse.Capture(TabList);
```
To:
```csharp
Mouse.Capture(TabList, CaptureMode.SubTree);
```

`CaptureMode.Element` (the default) routes ALL mouse events exclusively to TabList, which means `TabItem_PreviewMouseMove` (attached to ListBoxItem children) never fires after capture begins, so the ghost position never updates and the drag never completes via `TabItem_PreviewMouseLeftButtonUp`. `CaptureMode.SubTree` keeps events routing to children within TabList.

**Bug 2: Hover handlers fire during drag (lines ~477-514)**

Add `if (_isDragging) return;` guard as the FIRST line in both the `outerBorder.MouseEnter` and `outerBorder.MouseLeave` lambda handlers. This prevents hover effects (showing/hiding pin and close buttons, changing background) from firing while a tab is being dragged, which causes visual artifacts.

**Bug 3: LostMouseCapture re-entrancy (lines ~128-143)**

The `TabList.LostMouseCapture` handler calls `CompleteDrag()` which calls `Mouse.Capture(null)`. On some WPF versions, `Mouse.Capture(null)` can re-fire `LostMouseCapture`, causing re-entrant cleanup. Add a re-entrancy guard:

At the top of the LostMouseCapture handler, before `if (_isDragging)`, add:
```csharp
if (_isCompletingDrag) return;
```

Add a new field `private bool _isCompletingDrag;` near the other drag fields (around line ~55).

In `CompleteDrag()` method (line ~1521), wrap the body:
```csharp
private void CompleteDrag()
{
    if (!_isDragging) { ResetDragState(); return; }
    _isCompletingDrag = true;
    try
    {
        // ... existing body ...
    }
    finally
    {
        _isCompletingDrag = false;
    }
}
```
