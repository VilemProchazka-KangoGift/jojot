<objective>
Implement the three-tier session matching algorithm (GUID -> name -> index) that reconnects saved desktop sessions to live desktops after a reboot. This ensures notes appear on the correct desktop across restarts.

Purpose: VDSK-03 is the core feature that makes JoJot "remember" which notes belong to which desktop. Without this, all sessions would be orphaned after every reboot when Windows assigns new GUIDs.
Output: MatchSessionsAsync in VirtualDesktopService, session CRUD methods in DatabaseService, startup integration in App.xaml.cs
</objective>


2. **UpdateSessionAsync(string oldGuid, string newGuid, string? name, int? index)** — Updates a session's desktop_guid, desktop_name, and desktop_index:
```csharp
public static async Task UpdateSessionAsync(string oldGuid, string newGuid, string? name, int? index)
```
Implementation: `UPDATE app_state SET desktop_guid=@newGuid, desktop_name=@name, desktop_index=@index WHERE desktop_guid=@oldGuid;`
IMPORTANT: Also update notes.desktop_guid to keep the FK consistent: `UPDATE notes SET desktop_guid=@newGuid WHERE desktop_guid=@oldGuid;`
Use parameterized queries (not string interpolation) for ALL values to prevent SQL injection.

3. **CreateSessionAsync(string desktopGuid, string? desktopName, int? desktopIndex)** — Inserts a new app_state row:
```csharp
public static async Task CreateSessionAsync(string desktopGuid, string? desktopName, int? desktopIndex)
```
Implementation: `INSERT OR IGNORE INTO app_state (desktop_guid, desktop_name, desktop_index) VALUES (@guid, @name, @index);`
Use INSERT OR IGNORE because the desktop_guid column has a UNIQUE constraint — if a session already exists for this GUID, silently skip.

All three methods must use parameterized queries. Add a private helper method to DatabaseService for creating parameterized commands:
```csharp
private static SqliteCommand CreateCommand(string sql, params (string name, object? value)[] parameters)
{
    var cmd = _connection!.CreateCommand();
    cmd.CommandText = sql;
    foreach (var (name, value) in parameters)
        cmd.Parameters.AddWithValue(name, value ?? DBNull.Value);
    return cmd;
}
```

And a corresponding `ExecuteNonQueryWithParamsAsync` that uses the write lock:

  <action>
Add two new public methods to VirtualDesktopService:

**MatchSessionsAsync()** — The core three-tier matching algorithm:
```csharp
public static async Task MatchSessionsAsync()
```

Implementation:
1. If `!IsAvailable`: log "Session matching skipped (fallback mode)" and return. Ensure 'default' session exists via `DatabaseService.CreateSessionAsync("default", null, null)`.

2. Get live desktops: `var liveDesktops = GetAllDesktops();`
3. Get stored sessions: `var storedSessions = await DatabaseService.GetAllSessionsAsync();`

4. **Tier 1 — GUID match (exact)**:
   - For each stored session, check if its desktop_guid appears in liveDesktops (by matching `d.Id.ToString()` to session.DesktopGuid)
   - If match found: mark both as matched. Update session's desktop_name and desktop_index from the live desktop (they may have changed).
   - Call `DatabaseService.UpdateSessionAsync(session.DesktopGuid, session.DesktopGuid, liveDesktop.Name, liveDesktop.Index)` — same GUID, but updated name/index

5. **Tier 2 — Name match (skip if ambiguous)**:
   - For each unmatched session with a non-null, non-empty DesktopName:
   - Find live desktops (not yet matched) where `d.Name == session.DesktopName`
   - If exactly ONE match: reassign session to that desktop's GUID. Call `UpdateSessionAsync(oldGuid, newGuid, name, index)` — update ALL three fields per user decision.
   - If ZERO or 2+ matches: skip this session (fall through to Tier 3). Per user decision: ambiguous name match (multiple desktops share same name) skips name tier entirely.

6. **Tier 3 — Index match (strict condition)**:
   - Collect all unmatched sessions and all unmatched live desktops
   - For each unmatched session with a non-null DesktopIndex:
   - Find unmatched live desktops at that index
   - Apply the strict condition (VDSK-05): index matching only when EXACTLY one unmatched session and one unmatched desktop exist at that index
   - If strict condition met: reassign. Call `UpdateSessionAsync`.
   - Otherwise: session remains unmatched (orphaned)

7. After all tiers: log summary. Unmatched sessions are orphaned — do NOT delete them (preserved for Phase 8 recovery panel per user decision).

8. **Create sessions for new desktops**: For each live desktop that was NOT matched to any stored session, call `DatabaseService.CreateSessionAsync(id.ToString(), name, index)`.

Log each tier's results: "Tier 1 (GUID): {N} matched", "Tier 2 (Name): {N} matched", "Tier 3 (Index): {N} matched", "Orphaned: {N} sessions", "New: {N} desktops".
