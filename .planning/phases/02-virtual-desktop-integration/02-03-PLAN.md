**JoJot/Interop/VirtualDesktopNotificationListener.cs** — COM callback implementor:

Create a class that implements the IVirtualDesktopNotification COM callback interface. This class receives callbacks from Windows when desktops are renamed, switched, created, or destroyed.

```csharp
using System.Runtime.InteropServices;

namespace JoJot.Interop;

/// <summary>
/// COM callback implementor for virtual desktop notifications.
/// Windows calls these methods when desktop events occur.
/// CRITICAL: Do NOT call Release() on any parameters — Windows manages their lifetime.
/// </summary>
[ComVisible(true)]
internal class VirtualDesktopNotificationListener
{
    public event Action<Guid, string>? DesktopRenamed;
    public event Action<Guid, Guid>? CurrentDesktopChanged;
    public event Action<Guid>? DesktopCreated;
    public event Action<Guid>? DesktopDestroyed;
}
```

IMPORTANT: The exact method signatures on IVirtualDesktopNotification vary by build. The notification interface methods need to match the vtable layout exactly. For both 23H2 and 24H2, the key methods are:

For the notification callback, since the interface varies by build, implement it as a raw COM callable wrapper. The approach:

1. Define the class with `[ComVisible(true)]`
2. Use marshal-by-reference pattern so COM can call back into managed code
3. The class needs to expose methods that match the notification interface vtable
4. For VirtualDesktopRenamed: extract desktop GUID and new name, fire the DesktopRenamed event
5. For CurrentVirtualDesktopChanged: extract old and new desktop GUIDs, fire CurrentDesktopChanged event
6. All other notification methods (Created, DestroyBegin, DestroyFailed, Destroyed): return S_OK (0)

Since the exact vtable varies, use the simplest approach: implement via `ICustomQueryInterface` or a ComVisible class with the correct method order. The registration uses `IVirtualDesktopNotificationService.Register()` which takes an `IUnknown` pointer and returns a cookie (uint).

Actually, the SIMPLEST working approach for .NET COM callbacks with undocumented interfaces:

1. Define `IVirtualDesktopNotification` as a [ComImport] interface with the build-specific GUID
2. Implement it in `VirtualDesktopNotificationListener`
3. Register the listener with the notification service

Since the GUID varies by build, define TWO interface variants (one for 23H2, one for 24H2) OR define one interface and use dynamic COM registration. The simplest path: define the interface for one build and verify at runtime.

For now, start with the 24H2 variant since it's most current. Define:

internal interface IVirtualDesktopNotification

    void VirtualDesktopDestroyed(IntPtr monitors, IntPtr desktopDestroyed, IntPtr desktopFallback);
    void VirtualDesktopMoved(IntPtr monitors, IntPtr desktop, int oldIndex, int newIndex);
    void VirtualDesktopRenamed(IntPtr desktop, IntPtr newName);
    void ViewVirtualDesktopChanged(IntPtr view);
    void CurrentVirtualDesktopChanged(IntPtr monitors, IntPtr desktopOld, IntPtr desktopNew);
    void VirtualDesktopWallpaperChanged(IntPtr desktop, IntPtr path);

The `VirtualDesktopNotificationListener` class implements `IVirtualDesktopNotification`:
- VirtualDesktopRenamed: Extract desktop GUID from the IntPtr (via Marshal.QueryInterface for IVirtualDesktop, then GetId()). Extract name from HString IntPtr via `Marshal.PtrToStringUni` or HString helper. Fire `DesktopRenamed?.Invoke(desktopId, name)`.
- CurrentVirtualDesktopChanged: Extract GUIDs from both desktop pointers. Fire `CurrentDesktopChanged?.Invoke(oldId, newId)`.
- All other methods: no-op, return immediately. Do NOT call Release on any IntPtr parameters.

**JoJot/MainWindow.xaml.cs** — Add title management:

Add a public method `UpdateDesktopTitle(string? desktopName, int? desktopIndex)`:
```csharp
/// <summary>
/// Updates the window title based on the current desktop identity.
/// Title format (per user decision):
///   - "JoJot — {desktop name}" when name is known and non-empty
///   - "JoJot — Desktop N" when name is empty but index is known (N = index + 1)
///   - "JoJot" in fallback mode or when no desktop info available
/// Uses em-dash (—) with spaces, not hyphen (-) or en-dash (–).
/// </summary>
public void UpdateDesktopTitle(string? desktopName, int? desktopIndex)
{
    if (!string.IsNullOrEmpty(desktopName))
    {
        Title = $"JoJot \u2014 {desktopName}";  // \u2014 = em-dash
    }
    else if (desktopIndex.HasValue)
    {
        Title = $"JoJot \u2014 Desktop {desktopIndex.Value + 1}";
    }
    else
    {
        Title = "JoJot";
    }
}
```

NOTE: The em-dash character is U+2014 (—). Use the Unicode escape `\u2014` to ensure the correct character is used regardless of source file encoding.
