- `IVirtualDesktopManagerInternal_Base` — methods in vtable order: `GetCount(IntPtr hWndOrMon, out uint count)`, `MoveViewToDesktop(IntPtr view, IntPtr desktop)`, `CanViewMoveDesktops(IntPtr view, out bool canMove)`, `GetCurrentDesktop(IntPtr hWndOrMon, out IntPtr desktop)`, `GetDesktops(IntPtr hWndOrMon, out IntPtr desktops)`, `GetAdjacentDesktop(IntPtr desktop, int direction, out IntPtr adjacentDesktop)`, `SwitchDesktop(IntPtr hWndOrMon, IntPtr desktop)`, `CreateDesktop(IntPtr hWndOrMon, out IntPtr desktop)`, `MoveDesktop(IntPtr desktop, IntPtr hWndOrMon, int index)`, `RemoveDesktop(IntPtr desktop, IntPtr fallback)`, `FindDesktop(ref Guid desktopId, out IntPtr desktop)`, `GetDesktopSwitchIncludeExcludeViews(IntPtr desktop, out IntPtr views1, out IntPtr views2)`, `SetDesktopName(IntPtr desktop, IntPtr name)`, `SetDesktopWallpaper(IntPtr desktop, IntPtr path)`, `UpdateWallpaperPathForAllDesktops(IntPtr path)`, `CopyDesktopState(IntPtr src, IntPtr dst)`, `GetDesktopIsPerMonitor(out bool perMonitor)`, `SetDesktopIsPerMonitor(bool perMonitor)`

**JoJot/Interop/VirtualDesktopInterop.cs** — Low-level COM activation:
- `internal static class VirtualDesktopInterop`
- Namespace: `JoJot.Interop`
- Private fields: `IVirtualDesktopManager? _manager`, `IntPtr _managerInternal` (raw IntPtr for undocumented interface), `IntPtr _notificationService`, `GuidSet? _guidSet`
- `Initialize()`:
  1. Get build number via `Environment.OSVersion.Version.Build`
  2. Call `ComGuids.Resolve(buildNumber)` — if null, throw `PlatformNotSupportedException($"Unsupported OS build {buildNumber}")`
  3. Store `_guidSet`
  4. Create IVirtualDesktopManager via `Activator.CreateInstance(Type.GetTypeFromCLSID(ComGuids.CLSID_VirtualDesktopManager)!)` and cast
  5. Create ImmersiveShell via `Activator.CreateInstance(Type.GetTypeFromCLSID(ComGuids.CLSID_ImmersiveShell)!)` and cast to IServiceProvider10
  6. QueryService for VirtualDesktopManagerInternal: `provider.QueryService(ref CLSID_VirtualDesktopManagerInternal, ref _guidSet.IVirtualDesktopManagerInternal, out _managerInternal)`
  7. QueryService for NotificationService (same pattern) — store the IntPtr for Plan 02-03 to use
- `GetWindowDesktopId(IntPtr hwnd)`: Call `_manager!.GetWindowDesktopId(hwnd, out var id)` and return `id`
- `GetCurrentDesktop()`: Call through _managerInternal IntPtr using `Marshal.QueryInterface` and vtable dispatch — return `(Guid id, string name)` tuple. For getting name: after getting the IVirtualDesktop pointer, use `Marshal.QueryInterface` with the IVirtualDesktop GUID, then call GetId() and GetName() (vtable index 1 and 2 after IsViewVisible at index 0)
- `GetAllDesktops()`: Call GetDesktops on the internal manager, iterate the IObjectArray, for each extract Id, Name, and the iteration index. Return `List<(Guid id, string name, int index)>`
- `Dispose()`: Release all COM objects via `Marshal.ReleaseComObject` if non-null, set to null

IMPORTANT: All calls to undocumented COM interfaces must go through `Marshal.GetDelegateForFunctionPointer` with the correct vtable offsets. Since the IVirtualDesktopManagerInternal interface is NOT defined with [ComImport] (the GUID varies), use raw IntPtr + vtable dispatch. Define delegate types for each method signature needed.

Alternative simpler approach (PREFERRED): Since we know the method signatures, define the interface WITH [ComImport] but using a placeholder GUID. At runtime, use `Marshal.QueryInterface` with the correct build-specific GUID to get the correctly-typed pointer. The vtable layout is the same across builds — only the IID changes. So define:

```csharp
// Use 24H2 GUID as default — will be overridden at runtime via QueryInterface
[ComImport]
[InterfaceType(ComInterfaceType.InterfaceIsIUnknown)]
[Guid("53F5CA0B-158F-4124-900C-057158060B27")]
internal interface IVirtualDesktopManagerInternal { ... }
```

And in Initialize(), verify the interface is obtainable with the build-specific GUID. If the GUID doesn't match the compiled attribute, use `Marshal.QueryInterface(comObject, ref correctGuid, out ptr)` then `Marshal.GetObjectForIUnknown(ptr)` and cast.

**JoJot/Services/VirtualDesktopService.cs** — Public service boundary:
- `public static class VirtualDesktopService`
- Namespace: `JoJot.Services`
- Properties: `bool IsAvailable { get; }`, `string CurrentDesktopGuid { get; }`, `string CurrentDesktopName { get; }`
- `InitializeAsync()`:
  1. Wrap all COM init in try/catch(Exception)
  2. On success: set `_isAvailable = true`, call `GetCurrentDesktop()` to populate initial state
  3. On failure: log with `LogService.Warn($"Virtual desktop API unavailable — fallback mode: {ex.Message}")`, set `_isAvailable = false`, `_currentDesktopGuid = "default"`, `_currentDesktopName = ""`
  4. Always log: `LogService.Info($"VirtualDesktopService: available={_isAvailable}, desktop={_currentDesktopGuid}")`
- `GetAllDesktops()`: returns `IReadOnlyList<DesktopInfo>`. If not available, return single-element list with `new DesktopInfo(Guid.Empty, "", 0)`.
- `GetCurrentDesktopInfo()`: returns `DesktopInfo`. If not available, return `new DesktopInfo(Guid.Empty, "", 0)`.
- `Shutdown()`: Dispose interop. Called from App.OnExit.

COM initialization MUST happen on the WPF Dispatcher thread (STA requirement). The InitializeAsync method should verify it's on the UI thread or throw if not.

Per user decision on logging granularity (Claude's Discretion): Log COM initialization failure once (in InitializeAsync). Do NOT log every individual COM call failure — just return fallback defaults silently. This prevents log spam during extended fallback mode.
