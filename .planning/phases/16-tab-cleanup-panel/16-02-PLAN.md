**Replace the stub `CleanupDelete_Click` method** (created in plan 01) with the full implementation:

```csharp
private void CleanupDelete_Click(object sender, MouseButtonEventArgs e)
{
    var candidates = GetCleanupCandidates();
    if (candidates.Count == 0) return;

    // Build confirmation message
    int pinnedCount = candidates.Count(t => t.Pinned);
    string pinnedNote = pinnedCount > 0 ? $" (including {pinnedCount} pinned)" : "";
    string message = $"This will permanently delete {candidates.Count} tab{(candidates.Count == 1 ? "" : "s")}{pinnedNote}. This cannot be undone.";

    ShowConfirmation(
        "Clean up tabs",
        message,
        () => _ = ExecuteCleanupDeleteAsync(candidates)
    );
}
```

**Add ExecuteCleanupDeleteAsync method** — performs hard-delete (not soft-delete, per CONTEXT.md decision). This differs from `DeleteMultipleAsync` which uses soft-delete with toast/undo. Cleanup deletion is permanent immediately after confirmation:

```csharp
private async Task ExecuteCleanupDeleteAsync(List<NoteTab> candidates)
{
    // Save current content before deletion
    SaveCurrentTabContent();

    // Commit any existing pending soft-deletion first (from single-tab delete)
    await CommitPendingDeletionAsync();

    bool wasActiveDeleted = _activeTab != null && candidates.Any(t => t.Id == _activeTab.Id);
    int activeOriginalIndex = wasActiveDeleted ? _tabs.IndexOf(_activeTab!) : 0;

    // Remove from in-memory collection
    foreach (var tab in candidates)
        _tabs.Remove(tab);

    // Hard-delete from database (permanent, no undo)
    foreach (var tab in candidates)
        await DatabaseService.DeleteNoteAsync(tab.Id);

    // Rebuild tab list UI
    RebuildTabList();

    // Handle active tab fallback
    if (wasActiveDeleted)
    {
        if (_tabs.Count > 0)
        {
            await ApplyFocusCascadeAsync(activeOriginalIndex);
        }
        else
        {
            // No tabs survive — create a new empty tab
            await CreateNewTabAsync();
        }
    }

    // Refresh the cleanup panel preview list (panel stays open)
    if (_cleanupPanelOpen)
    {
        RefreshCleanupPreview();
    }
}
```

**IMPORTANT implementation details:**
- Do NOT use `DeleteMultipleAsync` — that method uses soft-delete with toast and undo, which violates the CONTEXT.md decision that cleanup deletion is permanent after confirmation.
- The `SaveCurrentTabContent()` call ensures the active tab's content is saved before any removal.
- `CommitPendingDeletionAsync()` clears any prior soft-deletion that might still be pending (from a single-tab delete with the X button).
- `CreateNewTabAsync` is an existing method in the codebase — verify it exists. If it doesn't exist by that name, look for the tab creation logic used when there are no tabs. It may be named differently (e.g., `CreateTabAsync`, `AddNewTab`, or similar).
