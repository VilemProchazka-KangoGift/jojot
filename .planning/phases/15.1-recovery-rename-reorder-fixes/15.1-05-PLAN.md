<objective>
Fix WPF animation local-value reversion bug that causes dragged tabs to snap back to 50% opacity after the fade-in animation completes, and add _isDragging guards to pin/close button hover handlers to suppress visual artifacts during drag.

Purpose: UAT tests 1 and 2 both failed because tabs stay faded after drop. Root cause is that line 1591 sets `item.Opacity = 0.5` as a WPF local value, and when `BeginAnimation(null)` clears the animation clock in the Completed handler, WPF reverts to that local value instead of 1.0.

Output: Working drag-to-reorder with correct opacity behavior throughout the drag lifecycle.

Three changes in JoJot/MainWindow.xaml.cs:

**Change 1: Remove the local Opacity assignment in CompleteDrag (line 1591)**

In the CompleteDrag method, find the line (currently ~line 1591):
```csharp
item.Opacity = 0.5;
```
DELETE this line entirely. The `From = 0.5` parameter on the DoubleAnimation (line 1594) already handles setting the initial animation value. By removing the local value set, when the animation clock is cleared via `BeginAnimation(null)` in the Completed handler, WPF will revert to the default Opacity of 1.0 instead of the stale local value of 0.5.

Do NOT change anything else in the fade-in animation block (lines 1592-1600). The `From = 0.5, To = 1.0` DoubleAnimation and the `BeginAnimation(null)` cleanup in Completed are correct.
