  <action>
Create README.md at the repo root (P:\projects\JoJot\README.md) with the following structure and content. This is a marketing document, not technical documentation.

**Structure (top to bottom):**

1. **Header area** — App icon (resources/icon-jj-3.png) displayed at a reasonable size (64-96px), followed by the app name "JoJot". No GitHub badges. Use an HTML img tag for size control.

2. **Tagline/hook** — One punchy line centering on "notes follow your virtual desktops" as the unique angle. This is the lead differentiator no other notepad offers. Keep it short and confident.

3. **Screenshot placeholder** — An HTML comment marking where a screenshot will go later, e.g. `<!-- Screenshot: Add hero screenshot here -->`. Structure the README so a screenshot fits naturally in this spot.

4. **Feature highlights** — Simple markdown bullet list with exactly these 6 features in this order. Each feature: **bold title** + one-line description. Keep descriptions tight and scannable.
   - **Virtual desktop notes** — each desktop gets its own set of notes (lead differentiator)
   - **Tabbed editing** — multiple notes per window, pin tabs, drag to reorder
   - **No save, no naming** — never asks you to save or name things, just type
   - **Auto-save** — everything persists automatically
   - **Super fast, no bloat** — lightweight, instant startup
   - **Easy cleanup** — clean up old notes easily

   Do NOT mention secondary features (hotkey, themes, find & replace, font sizes, etc.). Only these 6.

5. **Install section** — Short heading + single download link pointing to `https://github.com/vproc/JoJot/releases`. No build-from-source instructions. No system requirements section. Just the download link.

6. **License footer** — One-line MIT License mention at the bottom. Keep it minimal (e.g., "MIT License" with a link to LICENSE file).

**Tone guidelines:**
- Clean & confident — professional but not corporate
- Short sentences, bold claims, no fluff
- Think Linear/Raycast README style
- No AI credit — focus purely on the product
- No comparison to other tools
- No keyboard shortcuts reference

**Technical details:**
- Icon path: `resources/icon-jj-3.png` (relative from repo root)
- Use standard markdown with minimal HTML (only for icon sizing)
- The hook should make a visitor immediately understand what makes JoJot different
- "Never asks you to save or name things" is a key selling point per user — make it land
- "No bloat" and "super fast" are product values to convey, not just features
