# Milestone v1.0: MVP

**Status:** SHIPPED 2026-03-03
**Phases:** 1-10 (14 total including 4 decimal gap-closure phases)
**Total Plans:** 31

## Overview

JoJot v1.0 MVP — a lightweight WPF desktop app for plain-text notes tied to Windows virtual desktops. Each virtual desktop gets its own independent JoJot window with its own tabs and saved state. One background process manages all windows, accessible from the taskbar. Built for fastest possible capture — no formatting, no cloud sync, no clutter.

**Key Decisions:**
- PublishReadyToRun instead of Native AOT (WPF incompatible with PublishAot)
- Custom two-tier undo/redo (WPF native TextBox undo clears on tab switch)
- Single process + named pipe IPC (one SQLite connection, consistent state)
- Content inspection for file drop (not extension-based)
- Three-tier session matching (GUIDs reassigned on reboot)
- No confirmation dialogs for delete (4-second undo toast instead)
- Raw COM interop with [ComImport] (no NuGet packages for .NET 10)
- Custom Popup for menus (WPF ContextMenu can't use DynamicResource easily)

**Issues Resolved:**
- Virtual desktop COM GUIDs differ between Windows 11 23H2 and 24H2 — resolved with build-specific GUID dictionary
- RegisterHotKey and SetForegroundWindow — resolved with HwndSource message hook
- ORPH-03 Open button missing — resolved in Phase 8.1
- EDIT-03 FlushAsync on tab switch — resolved in Phase 8.1
- OnClosing missing deletion commit — resolved in Phase 10.1
- FlushAsync before drag detection — resolved in Phase 10.1
