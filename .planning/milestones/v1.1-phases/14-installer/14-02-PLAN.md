<task type="auto">
  <name>Task 1: Install Inno Setup, publish app, and compile installer</name>
  <files></files>
  <action>
1. **Install Inno Setup** if not already present:
   ```bash
   # Check if ISCC.exe exists
   ls "C:/Program Files (x86)/Inno Setup 6/ISCC.exe" 2>/dev/null
   # If not found, install via winget:
   winget install JRSoftware.InnoSetup --accept-package-agreements --accept-source-agreements
   ```
   After install, verify ISCC.exe exists at `C:\Program Files (x86)\Inno Setup 6\ISCC.exe`.

2. **Publish the application** (self-contained, win-x64):
   ```bash
   dotnet publish JoJot/JoJot.csproj -c Release -r win-x64 --self-contained
   ```
   Verify the publish output directory exists and contains JoJot.exe:
   `JoJot/bin/Release/net10.0-windows/win-x64/publish/JoJot.exe`

3. **Create output directory** if needed:
   ```bash
   mkdir -p installer/output
   ```

4. **Compile the Inno Setup script:**
   ```bash
   "C:/Program Files (x86)/Inno Setup 6/ISCC.exe" installer/jojot.iss
   ```
   This should produce `installer/output/JoJot-2026.3.0-Setup.exe`.

5. **Verify the output:**
   - Check the file exists and has a reasonable size (should be 50-150 MB for a self-contained .NET app)
   - Print the file size for the summary

6. **Add installer/output/ to .gitignore** if not already there -- installer output should not be committed. Also add `JoJot/bin/` and `JoJot/obj/` if not already ignored.
  </action>
  <verify>
    <automated>cd P:/projects/JoJot && test -f installer/output/JoJot-2026.3.0-Setup.exe && echo "INSTALLER BUILT SUCCESSFULLY" && ls -lh installer/output/JoJot-2026.3.0-Setup.exe</automated>
  </verify>
  <done>installer/output/JoJot-2026.3.0-Setup.exe exists as a valid Inno Setup installer. File size is reasonable for a self-contained .NET 10 WPF app (50-150 MB).</done>
</task>

<task type="checkpoint:human-verify" gate="blocking">
  <what-built>Complete JoJot installer package -- a self-contained Windows installer that bundles .NET 10 runtime, installs to Program Files, creates Start Menu shortcut, and includes auto-start option.</what-built>
  <how-to-verify>
1. **Run the installer:**
   - Double-click `installer/output/JoJot-2026.3.0-Setup.exe`
   - Verify the wizard shows: Welcome page -> Progress -> Finish (no license, no path picker, no component selection)
   - Verify the "Launch JoJot when Windows starts" checkbox appears and is UNCHECKED by default
   - Verify the Finish page shows "Launch JoJot" checkbox, CHECKED by default
   - Leave "Launch JoJot" checked and click Finish

2. **Verify the installed app:**
   - JoJot should launch from the installed location (C:\Program Files\JoJot\)
   - Verify the app works normally: create a tab, type text, switch tabs, check themes
   - Verify the EXE has the custom icon (check taskbar icon, Alt+Tab icon)
   - Check Start Menu: "JoJot" shortcut should exist under a "JoJot" group
   - Check there is NO desktop shortcut

3. **Verify version metadata:**
   - Right-click JoJot.exe in Program Files -> Properties -> Details tab
   - Product version should show "2026.3.0"
   - Company should show "Vilem Prochazka"
   - Product name should show "JoJot"
   - File description should show "Per-virtual-desktop notepad for Windows"

4. **Test uninstall:**
   - Close JoJot
   - Go to Settings -> Apps -> Installed apps -> find "JoJot" -> Uninstall
   - OR run the uninstaller from Start Menu
   - Verify the prompt appears: "Delete your JoJot data?" with default NO
   - Click No -- verify program files are removed but %LocalAppData%\JoJot\ still exists
   - (Optional: reinstall, then uninstall again choosing Yes to verify data deletion)

5. **Test upgrade scenario (optional):**
   - Install the app, run it, create a note with some text
   - Run the installer again while JoJot is still running
   - Verify JoJot is force-closed and the install proceeds
   - Launch JoJot after upgrade -- verify the note text is preserved
  </how-to-verify>
  <resume-signal>Type "approved" if the installer works correctly, or describe any issues found.</resume-signal>
</task>
