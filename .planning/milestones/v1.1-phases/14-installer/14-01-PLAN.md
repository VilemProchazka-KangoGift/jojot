<task type="auto">
  <name>Task 1: Add version metadata and create app icon</name>
  <files>JoJot/JoJot.csproj, JoJot/Assets/jojot.ico</files>
  <action>
1. **Modify JoJot/JoJot.csproj** -- add a new PropertyGroup (or extend the existing one) with these properties:
   - `<Version>2026.3.0</Version>` (CalVer)
   - `<AssemblyVersion>2026.3.0.0</AssemblyVersion>`
   - `<FileVersion>2026.3.0.0</FileVersion>`
   - `<Company>Vilem Prochazka</Company>` (use ASCII-safe version; diacritics in Company can cause encoding issues in PE metadata)
   - `<Authors>Vilem Prochazka</Authors>`
   - `<Product>JoJot</Product>`
   - `<Description>Per-virtual-desktop notepad for Windows</Description>`
   - `<Copyright>Copyright 2026 Vilem Prochazka</Copyright>`
   - `<ApplicationIcon>Assets\jojot.ico</ApplicationIcon>`

   Keep all existing PropertyGroup content (OutputType, TargetFramework, etc.) intact. Add the new properties to the existing PropertyGroup or create a second one -- either is fine.

2. **Create JoJot/Assets/ directory** if it does not exist.

3. **Create JoJot/Assets/jojot.ico** -- a multi-resolution icon file. Use a PowerShell or C# script to programmatically generate the icon:
   - Create a simple "J" lettermark design: a rounded-corner colored square background (#4A90D9 blue) with a white bold "J" letter centered
   - Generate images at 16x16, 32x32, 48x48, and 256x256 pixel sizes
   - Combine into a single .ico file with all four resolutions
   - Use System.Drawing (available in .NET) to create the bitmaps and write the ICO binary format
   - The ICO file format: 6-byte header (reserved=0, type=1, count=N), then N 16-byte directory entries, then raw bitmap data for each size
   - For 256x256, use PNG compression (standard for large ICO sizes); for smaller sizes, use uncompressed BMP data or PNG (PNG is simpler and widely supported in ICO for all sizes on Windows Vista+)

   Write a temporary C# script, run it with `dotnet script` or `dotnet run` via a temporary project, then delete the script. Alternatively, use PowerShell with Add-Type to access System.Drawing.

4. **Verify** the build still succeeds with the new icon and metadata by running `dotnet build JoJot/JoJot.slnx`.
  </action>

**[Code] section (Pascal Script):**
Implement the uninstall data-deletion prompt:
```pascal
procedure CurUninstallStepChanged(CurUninstallStep: TUninstallStep);
var
  DataDir: string;
begin
  if CurUninstallStep = usPostUninstall then
  begin
    DataDir := ExpandConstant('{localappdata}\JoJot');
    if DirExists(DataDir) then
    begin
      if MsgBox('Delete your JoJot data (notes and preferences)?'#13#10#13#10'If you choose No, your data will be preserved for future installations.',
                mbConfirmation, MB_YESNO or MB_DEFBUTTON2) = IDYES then
      begin
        DelTree(DataDir, True, True, True);
      end;
    end;
  end;
end;
```
The default button is No (MB_DEFBUTTON2) per user decision -- data preserved by default.
