This task fixes UAT Tests 4 and 5 with three targeted changes in the `CreateRecoveryRow` method of `JoJot/MainWindow.xaml.cs`.

**Change 1 — Remove horizontal margin from row container** (around line 2515):
Change:
```csharp
Margin = new Thickness(12, 10, 12, 10)
```
To:
```csharp
Margin = new Thickness(0, 10, 0, 10)
```
Why: The ScrollViewer in XAML already has `Margin="16,0,16,16"` providing 16px horizontal padding. The container's 12px horizontal margin was compounding to 28px total, misaligning rows with the "Recover Sessions" header (which has only the 16px from its Grid Margin).

**Change 2 — Make desktop name bold and larger** (around lines 2522-2523):
Change:
```csharp
FontSize = 13,
FontWeight = FontWeights.SemiBold,
```
To:
```csharp
FontSize = 14,
FontWeight = FontWeights.Bold,
```
Why: User reported "no desktop name" — SemiBold at 13pt was not visually distinct enough from the 11pt metadata text. Bold at 14pt creates clear visual hierarchy.

**Change 3 — Remove horizontal margin from divider** (around line 2665):
Change:
```csharp
Margin = new Thickness(12, 0, 12, 0)
```
To:
```csharp
Margin = new Thickness(0, 0, 0, 0)
```
Why: Same compounding issue as the container margin. The divider should span the full width within the ScrollViewer's existing 16px padding.

**Do NOT change** the ScrollViewer XAML margin, the tab preview lines, the button layout, or any other part of CreateRecoveryRow. These three changes are the complete fix.
