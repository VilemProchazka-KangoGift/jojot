## Tests

### 1. Tab drag shows 50% opacity during drag
expected: Click and drag a tab to reorder. The dragged tab should fade to 50% opacity in its original position and STAY faded while you hold the mouse button. No flash-and-snap-back to full opacity.
result: pass
note: Fixed by blocking SelectionChanged during drag, moving drag start to TabList-level handler, and targeting content Border opacity instead of ListBoxItem opacity.

### 2. Drop restores full opacity
expected: After dragging a tab and releasing, the tab should animate from 50% back to 100% opacity (~200ms fade-in). It should NOT stay faded after dropping.
result: issue
reported: "jumps to full opacity, no animation"
severity: cosmetic
note: Animation code runs (confirmed via logs) but RebuildTabList visual reset masks the subtle opacity transition. Tab does restore to full opacity correctly.

### 3. No hover effects on pin/close during drag
expected: While dragging a tab, hover over pin/close buttons on other tabs. No hover visual effects (color changes, highlights) should appear on those buttons during the drag.
result: pass
note: Fixed by blocking SelectionChanged during drag which was the real cause of visual artifacts.

### 4. Recovery panel tab preview alignment
expected: Open recovery panel. Tab preview lines (individual tab names with content excerpts) should be flush-aligned with the desktop name header — no extra left indentation.
result: pass

### 5. Recovery row shows desktop name
expected: Each recovery row should show the actual desktop name in bold. It should NOT say "Unknown desktop" if the desktop had a name in Windows.
result: pass
note: Fixed empty string fallback (DB returns '' not null). Shows "Unknown desktop" for orphaned GUIDs not in registry. Actual desktop names will show when DB stores non-empty names.
