### Human Verification Required

### 1. Recovery Panel Visual Layout

**Test:** Open a window with orphaned sessions from other desktops, click "Recover Sessions" in the hamburger menu.
**Expected:** Recovery panel shows flat full-width rows (not cards with borders/rounded corners). Each row has desktop name in bold (14pt), tab count and date in muted text (11pt), individual tab lines with name and excerpt flush-aligned with the header, Adopt button on left, red Delete button on right, with subtle divider lines between rows. Rows should be flush-aligned with the "Recover Sessions" header (no extra indentation).
**Why human:** Visual layout, spacing, font sizing, and color accuracy cannot be verified programmatically.

### 2. Escape-to-Cancel Rename

**Test:** Double-click a tab to start renaming, type some different text, press Escape once.
**Expected:** Rename is cancelled immediately on first press, original title is restored (typed text discarded), and cursor returns to the editor.
**Why human:** Keyboard event tunnel behavior and focus routing need interactive verification.

### 3. Drag-to-Reorder Visual

**Test:** Drag a tab to a new position in the list.
**Expected:** The dragged tab fades to 50% opacity in place (no ghost following cursor). Drop indicator lines appear at valid positions. On release, the tab appears at its new position and smoothly fades from 50% to 100% opacity over ~150ms. Tab stays at 100% after animation completes (no snap-back to 50%).
**Why human:** Animation timing, opacity visual appearance, and overall drag feel need interactive testing.

### 4. "+N More" Overflow

**Test:** Create a desktop session with more than 5 tabs, then trigger recovery panel from another desktop.
**Expected:** Only 5 tab previews shown, followed by "+N more" in muted italic text.
**Why human:** Requires multi-desktop setup with specific tab counts.

### 5. Registry Desktop Name Fallback

**Test:** Trigger recovery panel with orphaned sessions whose desktop names are not stored in the database.
**Expected:** Recovery rows show actual desktop names from the Windows registry instead of "Unknown desktop".
**Why human:** Requires specific orphaned session state and registry entries.
