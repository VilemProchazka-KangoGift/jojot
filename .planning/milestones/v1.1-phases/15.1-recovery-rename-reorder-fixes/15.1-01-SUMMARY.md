## Accomplishments
- Escape during tab rename now cancels on first press (window-level PreviewKeyDown tunnel fires before panel handlers)
- DragAdorner inner class and all 95 lines of adorner-related code fully removed
- Tab drag now fades the dragged item to 50% opacity in-place (no floating ghost)
- Drop completion animates the tab from 50% to 100% opacity over 150ms with CubicEase easing

## Files Created/Modified

## Decisions Made
- Rename Escape handler placed after `_isDragOverlayActive` block but before `ConfirmationOverlay` -- this ensures drag overlay still blocks all input, while rename cancel takes priority over panel dismissals
- DragAdorner class fully removed rather than simplified -- the plan called for in-place fade which makes the adorner unnecessary
- `TabList_PreviewMouseMove_DragFallback` handler removed entirely since it only existed to track adorner position in empty space
- Fade-in animation calls `BeginAnimation(null)` on completion to clear the animation binding and allow future property sets
- `_isCompletingDrag` re-entrancy guard retained -- still needed for `Mouse.Capture(null)` re-fire prevention even without adorner
