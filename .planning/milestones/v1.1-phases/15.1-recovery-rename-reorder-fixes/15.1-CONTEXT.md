# Phase 15.1: Recovery Panel, Tab Rename & Reorder Fixes - Context

**Gathered:** 2026-03-06
**Status:** Ready for planning

<domain>
## Phase Boundary

Polish three specific interactions from Phase 15: redesign recovery panel layout from cards to full-width rows with tab content, fix Escape-to-cancel rename (double-press bug), and replace drag ghost adorner with in-place fade-out. No new features — purely visual/interaction refinements.

</domain>

<decisions>
## Implementation Decisions

### Recovery panel layout
- Replace card-based `Border` elements with flat full-width rows separated by subtle divider lines
- Each row shows: desktop name (bold), tab count + date (muted), then individual tab details below
- Tab details: show tab name + text excerpt (~50 chars) per line; if no custom name was set, show excerpt only
- Show up to 5 tabs per session; if more exist, show "+N more" muted line
- Adopt and Delete buttons remain — Adopt on left, Delete (red) on right, positioned within the row
- No border, no rounded corners, no card shadow — clean list style like a settings panel

### Tab rename Escape fix
- Bug: sometimes Escape must be pressed twice to cancel a rename
- Root cause likely: Window-level `PreviewKeyDown` catches Escape before the rename TextBox's handler (WPF preview events tunnel from root to leaf)
- Fix: check for `_activeRename != null` early in the window-level Escape handling and route to `CancelRename()`
- After cancel, return focus to the editor (not the tab list)

### Drag-to-reorder visuals
- Remove the `DragAdorner` ghost overlay entirely — no bitmap snapshot following the cursor
- Dragged tab fades to ~50% opacity in-place in the list
- Keep existing drop indicator lines as-is (horizontal lines between items, suppressed at no-change positions)
- On drop completion: fade the tab back from 50% to 100% opacity over ~150ms at its new position

### Claude's Discretion
- Exact excerpt truncation length and ellipsis style
- Divider line styling (color, thickness) for recovery rows
- Fade-in easing curve for drop animation
- Any cleanup of now-unused `DragAdorner` class

</decisions>
