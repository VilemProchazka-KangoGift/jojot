Fix two bugs: unknown desktop name in move overlay and stale session metadata after reparenting.

**Bug 1 — "Unknown desktop" in move overlay (ShowDragOverlayAsync, line ~3543-3552):**

The COM API `GetName()` returns empty string for desktops the user hasn't renamed. Windows generates "Desktop N" labels client-side in Task View, not via COM. ShowDragOverlayAsync currently shows "From: Unknown desktop" when `GetDesktopNameAsync` returns null/empty.

Add an index-based fallback matching the pattern already used in `UpdateDesktopTitle`. Replace lines ~3543-3552 with:

```csharp
// R2-MOVE-01: Show source desktop name with index fallback
string sourceLabel;
try
{
    var sourceName = await DatabaseService.GetDesktopNameAsync(_desktopGuid);
    if (!string.IsNullOrEmpty(sourceName))
    {
        sourceLabel = sourceName;
    }
    else
    {
        // COM returns empty for un-renamed desktops; generate "Desktop N" like Windows does
        var desktops = VirtualDesktopService.GetAllDesktops();
        var sourceDesktop = desktops.FirstOrDefault(d =>
            d.Id.ToString().Equals(_desktopGuid, StringComparison.OrdinalIgnoreCase));
        sourceLabel = sourceDesktop != null
            ? $"Desktop {sourceDesktop.Index + 1}"
            : "Unknown desktop";
    }
}
catch
{
    sourceLabel = "Unknown desktop"; // best-effort
}
DragOverlaySourceName.Text = $"From: {sourceLabel}";
```

Also apply the same fallback to the `toName`/`displayName` logic (line ~3555). Currently empty toName shows "another desktop". Since `toName` comes from the COM name which may be empty for un-renamed desktops, add an index-based fallback:

```csharp
// Configure overlay content with name fallback
string displayName;
if (!string.IsNullOrEmpty(toName))
{
    displayName = toName;
}
else
{
    var desktops = VirtualDesktopService.GetAllDesktops();
    var targetDesktop = desktops.FirstOrDefault(d =>
        d.Id.ToString().Equals(toGuid, StringComparison.OrdinalIgnoreCase));
    displayName = targetDesktop != null
        ? $"Desktop {targetDesktop.Index + 1}"
        : "another desktop";
}
DragOverlayTitle.Text = $"Moved to {displayName}";
```

**Bug 2 — Session reparenting misses name/index (DragKeepHere_Click, line ~3616 + DatabaseService):**

`DragKeepHere_Click` calls `UpdateSessionDesktopGuidAsync` which only does `UPDATE app_state SET desktop_guid=@new`. This leaves the old `desktop_name` and `desktop_index` in the row. On restart, Tier 3 matching sees the stale index and silently reassigns the session to a remaining desktop at that index, consuming it instead of marking it as orphaned.

Replace the call to `UpdateSessionDesktopGuidAsync` in `DragKeepHere_Click` (line ~3616) with a call to the existing `UpdateSessionAsync` which updates all three columns plus the notes FK:

```csharp
// Update app_state session with full metadata (guid + name + index)
string targetName = targetInfo?.Name ?? "";
int? targetIndex = targetInfo?.Index;
await DatabaseService.UpdateSessionAsync(oldGuid, newGuid, targetName, targetIndex);
```

The `targetInfo` variable is already available (computed at line ~3611-3612). This replaces the `UpdateSessionDesktopGuidAsync(oldGuid, newGuid)` call.

Note: `UpdateSessionAsync` also migrates notes FK (desktop_guid in notes table), but `MigrateNotesDesktopGuidAsync` is already called at line ~3599. This is fine -- the UPDATE on notes at line 3599 sets them to newGuid, and `UpdateSessionAsync`'s notes UPDATE is a no-op (old=oldGuid matches nothing since they're already newGuid). However, to be clean, we can skip the double-update. Since `UpdateSessionAsync` handles both app_state AND notes, remove the separate `MigrateNotesDesktopGuidAsync` call at line ~3599 and let `UpdateSessionAsync` handle everything. OR keep both calls (the second is idempotent). Keeping both is safer and clearer -- leave `MigrateNotesDesktopGuidAsync` in place.

After this fix, `UpdateSessionDesktopGuidAsync` is no longer called from anywhere. Check if it's called elsewhere in the codebase. If not, leave it in place (dead code removal is optional polish, not a bug fix).
