**Fix 1 — Tab hover height (Gap 1):**
In CreateTabListItem(), immediately after `var row0 = new Grid();` (line ~323), add:
```csharp
row0.MinHeight = 22;
```
This ensures the row is always at least 22px (matching the pin/close Border containers), so toggling icon Visibility between Collapsed and Visible no longer causes vertical growth. The title TextBlock at FontSize 13 (~17-18px) fits within 22px.

**Fix 2 — Drag ghost in empty space (Gap 2):**
Add a PreviewMouseMove handler on TabList for ghost position updates when the mouse is between ListBoxItems (empty space where no TabItem_PreviewMouseMove fires).

In the constructor or InitializeComponent area where TabList event handlers are wired up, add:
```csharp
TabList.PreviewMouseMove += TabList_PreviewMouseMove_DragFallback;
```

Add the handler method near the existing drag code (~line 1385):
```csharp
/// <summary>
/// Fallback handler: updates drag ghost position when mouse is in empty space
/// between ListBoxItems (where TabItem_PreviewMouseMove doesn't fire).
/// </summary>
private void TabList_PreviewMouseMove_DragFallback(object sender, MouseEventArgs e)
{
    if (!_isDragging || _dragAdorner == null) return;
    _dragAdorner.UpdatePosition(e.GetPosition(TabList));
}
```

This handler fires for ALL mouse moves within the TabList (including empty space). When _isDragging is true and the mouse is between items, it keeps the ghost tracking the cursor. When the mouse IS over a ListBoxItem, both this and TabItem_PreviewMouseMove fire — the position update is idempotent so the double call is harmless.
