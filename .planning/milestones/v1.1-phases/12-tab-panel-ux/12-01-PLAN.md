**MainWindow.xaml.cs â€” `CreateTabListItem()` changes:**

1. **Remove left border, add rounded background highlight support:**
   Change `outerBorder` initialization:
   - `BorderThickness = new Thickness(0)` (was `2,0,0,0`)
   - Remove `BorderBrush` assignment entirely (no longer needed)
   - Add `CornerRadius = new CornerRadius(4)` for rounded selection highlight
   - Keep `Background = Brushes.Transparent` and `Padding = new Thickness(8, 6, 8, 6)`

2. **Replace pin emoji with Segoe Fluent Icons pin glyph:**
   Change the pinned-tab icon from:
   ```csharp
   Text = "\U0001F4CC",  // emoji pushpin
   FontSize = 11,
   ```
   To:
   ```csharp
   Text = "\uE718",  // Segoe Fluent Icons "Pin" glyph
   FontFamily = new System.Windows.Media.FontFamily("Segoe Fluent Icons, Segoe MDL2 Assets"),
   FontSize = 12,
   ```
   And add `SetResourceReference(TextBlock.ForegroundProperty, "c-text-muted")` to make it theme-aware.

3. **Adaptive title width when delete icon visible:**
   Replace the fixed `MaxWidth` on `labelBlock` with a column-based layout that adapts:
   - Change `row0` from a `StackPanel` to a `Grid` with column definitions:
     - Column 0: Auto (pin icon, if pinned)
     - Column 1: Star `*` (title â€” fills remaining space, with TextTrimming)
     - Column 2: Auto (delete icon area â€” 0 width when hidden, ~16px when visible)
   - Remove `MaxWidth` from `labelBlock` entirely (the Star column handles sizing)
   - Move the `deleteIcon` from the outer grid into row0's Column 2 (inside the row, not spanning both rows)
     - This way, when deleteIcon opacity is 0 (hidden), Column 2 collapses and title gets full width
     - When deleteIcon opacity is 1 (visible), Column 2 takes space and title shortens
   - To make the column actually collapse/expand:
     - In MouseEnter handler: set deleteIcon column width to `GridLength.Auto` and deleteIcon `Visibility = Visible`
     - In MouseLeave handler: set deleteIcon `Visibility = Collapsed` (this collapses Column 2 Auto back to 0)
     - Keep the opacity animation for smooth visual fade, but use Visibility to drive the column sizing
   - The `renameBox` should share Column 1 with the labelBlock (collapsed by default, shown on rename)
   - **Important:** Remove `Grid.SetRowSpan(deleteIcon, 2)` since delete icon moves into row0

4. **Update hover handlers on `outerBorder`:**
   MouseEnter handler:
   ```csharp
   outerBorder.MouseEnter += (s, e) =>
   {
       if (item != TabList.SelectedItem)
           outerBorder.Background = GetBrush("c-hover-bg");
       deleteIcon.Visibility = Visibility.Visible;
       AnimateOpacity(deleteIcon, 0, 1, 100);
   };
   ```
   MouseLeave handler:
   ```csharp
   outerBorder.MouseLeave += (s, e) =>
   {
       if (item != TabList.SelectedItem)
           outerBorder.Background = Brushes.Transparent;
       AnimateOpacity(deleteIcon, 1, 0, 100);
       // After fade-out completes, collapse to reclaim space
       var timer = new System.Windows.Threading.DispatcherTimer
       {
           Interval = TimeSpan.FromMilliseconds(100)
       };
       timer.Tick += (_, _) =>
       {
           timer.Stop();
           if (deleteIcon.Opacity == 0)
               deleteIcon.Visibility = Visibility.Collapsed;
       };
       timer.Start();
   };
   ```
