## Tech Debt Summary

### 1. SUMMARY Frontmatter Convention (All phases)
`requirements_completed` YAML field is empty across all 31 plan summaries. Requirements tracking relied on VERIFICATION.md + REQUIREMENTS.md traceability table instead. No functional impact.

### 2. GetPendingMovesAsync Write Lock (Phase 1)
`DatabaseService.GetPendingMovesAsync()` executes on `_connection` without `_writeLock`. Every other method uses the lock. Risk near-zero (called only at startup), but architectural inconsistency with DATA-02 design.

### 3. ShowDesktopCommand Dead Code (Phase 1)
Registered IPC message type with routed handler case, but handler only logs and breaks. No code path sends this command. Dead code inflating IPC protocol surface.

### 4. Phase 8 Human Testing (Phase 8)
VERIFICATION status `human_needed`. All 11 automated checks passed. 9 visual/interactive tests pending manual execution (hamburger menu, submenu, confirmation dialog, context menu, orphan badge, recovery panel, animations).
