Create `JoJot/Services/UndoStack.cs` — a per-tab in-memory undo/redo stack with two tiers.

**Class: UndoStack**

Fields:
- `private readonly List<string> _tier1` — fine-grained snapshots, max 50 (constant `MaxTier1 = 50`)
- `private readonly List<string> _tier2` — coarse checkpoints, max 20 (constant `MaxTier2 = 20`)
- `private int _currentIndex` — pointer into the combined logical sequence. Points to the current state. -1 means empty.
- `private DateTime _lastCheckpointTime` — tracks when last tier-2 checkpoint was created
- `public long TabId { get; }` — the NoteTab.Id this stack belongs to
- `public DateTime LastAccessTime { get; set; }` — updated on any push/undo/redo; used by UndoManager for LRU collapse

**Design for seamless two-tier traversal:**
The two tiers form a logical sequence: [tier-2 oldest ... tier-2 newest, tier-1 oldest ... tier-1 newest].
`_currentIndex` ranges from 0 to `_tier2.Count + _tier1.Count - 1`.
- Index 0 to `_tier2.Count - 1` = tier-2 entries
- Index `_tier2.Count` to `_tier2.Count + _tier1.Count - 1` = tier-1 entries

**Methods:**

`public void PushInitialContent(string content)`:
- Called once when a tab is loaded from DB. This is the "floor" — Ctrl+Z can always restore to this state.
- Clears both tiers, adds content as tier-1[0], sets `_currentIndex = 0`.
- Sets `_lastCheckpointTime = DateTime.Now`.

`public void PushSnapshot(string content)`:
- Called on every debounced autosave if content differs from current snapshot.
- If `_currentIndex` is not pointing to the end of the logical sequence, truncate everything after it (destroy redo future — linear stack per user decision).
- Add content to tier-1. If tier-1.Count > MaxTier1, remove oldest tier-1 entry (index 0 of _tier1).
- Update `_currentIndex` to point to the newly added entry.
- Update `LastAccessTime = DateTime.Now`.

`public void PushCheckpoint(string content)`:
- Called by the 5-minute checkpoint timer if content differs from the last checkpoint.
- Add content to tier-2. If tier-2.Count > MaxTier2, remove oldest tier-2 entry.
- Adjust `_currentIndex` to account for any removal.
- Update `_lastCheckpointTime = DateTime.Now`.

`public string? Undo()`:
- If `_currentIndex <= 0`, return null (nothing to undo).
- Decrement `_currentIndex`. Return the content at the new index position (resolving from tier-2 or tier-1 based on index range).
- Update `LastAccessTime`.

`public string? Redo()`:
- If `_currentIndex >= total count - 1`, return null (nothing to redo).
- Increment `_currentIndex`. Return the content at the new index position.
- Update `LastAccessTime`.

`public bool CanUndo => _currentIndex > 0`
`public bool CanRedo => _currentIndex < TotalCount - 1`
`private int TotalCount => _tier2.Count + _tier1.Count`

`public long EstimatedBytes`:
- Sum of `s.Length * 2` for all strings in both tiers (sizeof(char) = 2 in .NET).

`public void CollapseTier1IntoTier2()`:
- Take every 5th entry from tier-1 and add to tier-2 (respecting MaxTier2).
- Clear tier-1. Reset `_currentIndex` to point to end of tier-2.
- This is called by UndoManager during memory collapse.

`public void EvictOldestTier2(int count)`:
- Remove the oldest `count` entries from tier-2.
- Adjust `_currentIndex` accordingly.

`public bool ShouldCreateCheckpoint()`:
- Returns true if more than 5 minutes have elapsed since `_lastCheckpointTime`.

Private helper `private string GetContentAtIndex(int index)`:
- If index < _tier2.Count, return _tier2[index].
- Else return _tier1[index - _tier2.Count].

**Namespace:** `JoJot.Services`
**No external dependencies. Pure in-memory data structure.**

**File 1: JoJot/Services/UndoManager.cs**

Singleton that manages all UndoStacks across all tabs and enforces the 50MB global memory budget.

Fields:
- `private readonly Dictionary<long, UndoStack> _stacks` — maps NoteTab.Id to its UndoStack
- `private const long MaxBudgetBytes = 50L * 1024 * 1024` — 50MB hardcoded (UNDO-05)
- `private const double CollapseThreshold = 0.80` — trigger at 80%
- `private const double CollapseTarget = 0.60` — target 60%
- `private long? _activeTabId` — the currently active tab (never collapsed)

Methods:

`public static UndoManager Instance { get; }` — lazy singleton pattern (static readonly field or Lazy<T>).

`public UndoStack GetOrCreateStack(long tabId)`:
- If stack exists in dictionary, return it.
- Else create new UndoStack(tabId), add to dictionary, return it.

`public void SetActiveTab(long? tabId)`:
- Sets `_activeTabId`. Used during collapse to protect the active tab.

`public void RemoveStack(long tabId)`:
- Remove from dictionary. Called when tab is permanently deleted.

`public void PushSnapshot(long tabId, string content)`:
- Get or create stack for tabId. Call stack.PushSnapshot(content).
- After push, check if `TotalEstimatedBytes > MaxBudgetBytes * CollapseThreshold`. If so, call `CollapseOldest()`.

`public string? Undo(long tabId)`:
- Get stack, call Undo(). Return result.

`public string? Redo(long tabId)`:
- Get stack, call Redo(). Return result.

`public bool CanUndo(long tabId)` — delegates to stack.
`public bool CanRedo(long tabId)` — delegates to stack.

`public long TotalEstimatedBytes`:
- Sum of all stacks' EstimatedBytes.

`private void CollapseOldest()`:
- Get all stacks sorted by LastAccessTime ascending (oldest first).
- Skip the active tab (never collapse).
- For each stack (oldest first): call CollapseTier1IntoTier2(). If total is now below target, stop.
- If still above target: for each stack (oldest first, skip active): call EvictOldestTier2(5). If below target, stop.
- Log via LogService how much memory was freed.

`public UndoStack? GetStack(long tabId)`:
- Return stack if exists, null otherwise. Used for checkpoint checks.

**File 2: JoJot/Services/AutosaveService.cs**

Per-window autosave service using DispatcherTimer with reset-on-keystroke debounce.

Fields:
- `private readonly System.Windows.Threading.DispatcherTimer _timer`
- `private int _debounceMs = 500` — default, configurable via property
- `private DateTime _lastWriteCompleted = DateTime.MinValue`
- `private Func<(long TabId, string Content)>? _contentProvider` — lambda that returns current tab ID and editor content
- `private Action<long>? _onSaveCompleted` — callback after save completes (for UI refresh)

Constructor: `public AutosaveService()`
- Create DispatcherTimer with initial interval 500ms.
- Wire Tick event to internal handler.
- Do NOT start — Start is called when content changes.

Methods:

`public int DebounceMs { get; set; }` — updates timer interval.

`public void Configure(Func<(long TabId, string Content)> contentProvider, Action<long> onSaveCompleted)`:
- Sets the content provider and callback.

`public void NotifyTextChanged()`:
- Called by UI on every user-initiated TextChanged.
- EDIT-03 write frequency cap: Check if `DateTime.Now - _lastWriteCompleted < TimeSpan.FromMilliseconds(_debounceMs)`. If within cooldown AND timer is already running, return (don't reset — let existing timer fire).
- Otherwise: Stop timer, set interval to DebounceMs, Start timer.

`public async Task FlushAsync()`:
- Stop timer. If contentProvider returns content, save immediately via DatabaseService.UpdateNoteContentAsync.
- Called on tab switch and app close for synchronous flush.

`public void Stop()`:
- Stop the timer. Called on tab switch before switching context.

Private `async void OnTimerTick(object? sender, EventArgs e)`:
- Stop timer (single-shot).
- Get content from _contentProvider. If null or tab has no changes, return.
- Call DatabaseService.UpdateNoteContentAsync.
- Set `_lastWriteCompleted = DateTime.Now`.
- Push undo snapshot: `UndoManager.Instance.PushSnapshot(tabId, content)`.
- Call _onSaveCompleted callback.

**Namespace:** `JoJot.Services` for both files.
**Important:** AutosaveService does NOT directly reference MainWindow. It uses delegates/callbacks for loose coupling.
