<domain>
## Phase Boundary

Fix two specific gaps identified by the v1.0 milestone audit: the missing Open button in the orphan recovery panel (ORPH-03), and the autosave FlushAsync bypass on tab switch (EDIT-03). No new capabilities — these are targeted code corrections.

</domain>

<decisions>
## Implementation Decisions

### ORPH-03: Open button in recovery panel
- Add an Open button between Adopt and Delete in `CreateRecoveryCard` (MainWindow.xaml.cs:2039-2059)
- Use the existing `CreateCardButton` local function factory — non-destructive style (same as Adopt)
- Click handler calls `App.OpenWindowForOrphanAsync(guid)` then `RemoveOrphanGuid` + `RefreshAfterOrphanAction` (same pattern as Adopt/Delete)
- Button should NOT close the recovery panel — it refreshes like the other actions do

### EDIT-03: Autosave FlushAsync on tab switch
- In `TabList_SelectionChanged` (MainWindow.xaml.cs:420-433), replace `_autosaveService.Stop()` + `SaveCurrentTabContent()` with `await _autosaveService.FlushAsync()`
- `FlushAsync` already stops the timer internally and saves content through the service, keeping `_lastWriteCompleted` state accurate
- The `_checkpointTimer.Stop()` call should remain — checkpoint timer is separate from autosave
- Method signature needs to change from `void` to `async void` for the await (event handler — acceptable for WPF events)

### Claude's Discretion
- Whether to also fix the stale docstring on `MenuRecover_Click` (mentioned in audit as "Info" tech debt) — opportunistic cleanup if touched

</decisions>
