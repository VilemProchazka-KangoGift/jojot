1. Create JoJot/Services/StartupService.cs:
   - Public static class `StartupService` in namespace `JoJot.Services`
   - `CreateWelcomeTabIfFirstLaunch()`:
     - Query: `SELECT COUNT(*) FROM notes`
     - If count is 0 (first launch), insert a welcome note:
       - desktop_guid = "default" (Phase 1 has no virtual desktop detection yet)
       - name = "Welcome to JoJot"
       - content = brief welcome text (2-3 paragraphs covering: what JoJot is, that it ties notes to virtual desktops, basic shortcuts like Ctrl+T for new tab, Ctrl+W to close, that this tab can be deleted)
       - pinned = 0, sort_order = 0
     - Use `DatabaseService.ExecuteNonQueryAsync` with parameterized SQL
   - `RunBackgroundMigrationsAsync()`:
     - Calls `DatabaseService.RunPendingMigrationsAsync()` (the stub from Plan 01)
     - Wrapped in try/catch: on failure, `LogService.Error("Background migration failed", ex)` — never rethrow (per user decision: log and continue, retry next launch)

2. Update JoJot/App.xaml:
   - REMOVE `StartupUri="MainWindow.xaml"` attribute (we create the window manually in code)
   - ADD `Startup="OnAppStartup"` event handler instead
   - Keep ShutdownMode default in XAML (we set it in code to OnExplicitShutdown)

3. Rewrite JoJot/App.xaml.cs completely:
   - Remove existing usings (System.Configuration, System.Data) — not needed
   - Add: `using JoJot.Services;`, `using JoJot.Models;`, `using System.Diagnostics;`, `using System.IO;`
   - Private static fields:
     - `Mutex? _singleInstanceMutex` (prevents GC collection per research pitfall #1)
     - `CancellationTokenSource _appShutdownCts = new()`
     - `MainWindow? _mainWindow`
   - `async void OnAppStartup(object sender, StartupEventArgs e)`:
     - `var sw = Stopwatch.StartNew();`

     **Step 1: Initialize logging**
     - `string appDir = Path.Combine(Environment.GetFolderPath(Environment.SpecialFolder.LocalApplicationData), "JoJot");`
     - `Directory.CreateDirectory(appDir);`
     - `LogService.Initialize(appDir);`
     - `LogService.Info("JoJot starting...");`

     **Step 2: Mutex (PROC-01)**
     - `bool createdNew = IpcService.TryAcquireMutex(out var mutex);`
     - `_singleInstanceMutex = mutex;`
     - `GC.KeepAlive(_singleInstanceMutex);`
     - If NOT createdNew (second instance — PROC-03):
       - `LogService.Info("Second instance detected — sending activate command");`
       - `await IpcService.SendCommandAsync(new ActivateCommand());`
       - `Environment.Exit(0); return;`

     **Step 3: Set ShutdownMode (PROC-05)**
     - `ShutdownMode = ShutdownMode.OnExplicitShutdown;`

     **Step 4: Open database + schema (DATA-01, DATA-07, STRT-04)**
     - `string dbPath = Path.Combine(appDir, "jojot.db");`
     - `await DatabaseService.OpenAsync(dbPath);`
     - `await DatabaseService.EnsureSchemaAsync();`

     **Step 5: Integrity check + corruption recovery**
     - `bool healthy = await DatabaseService.VerifyIntegrityAsync();`
     - If NOT healthy: `await DatabaseService.HandleCorruptionAsync(dbPath);`

     **Step 6: Check pending_moves (stub for Phase 10 crash recovery)**
     - Comment: `// Phase 10: await DatabaseService.ResolvePendingMovesAsync();`
     - `LogService.Info("Pending moves check: skipped (Phase 10)");`

     **Step 7: Welcome tab on first launch**
     - `await StartupService.CreateWelcomeTabIfFirstLaunch();`

     **Step 8: Start IPC server (PROC-02)**
     - `IpcService.StartServer(HandleIpcCommand, _appShutdownCts.Token);`

     **Step 9: Create and show window**
     - `_mainWindow = new MainWindow();`
     - `_mainWindow.Show();`

     **Step 10: Log startup timing (STRT-01)**
     - `sw.Stop();`
     - `LogService.Info($"Startup complete in {sw.ElapsedMilliseconds}ms");`
     - `Debug.WriteLine($"[JoJot] Startup: {sw.ElapsedMilliseconds}ms");`

     **Step 11: Background migrations (STRT-03)**
     - `_ = Task.Run(() => StartupService.RunBackgroundMigrationsAsync());`

   - `HandleIpcCommand(IpcMessage message)`:
     - Switch on message type:
       - `ActivateCommand`: If `_mainWindow` is not null, call `_mainWindow.ActivateFromIpc()`. Log the activation.
       - `NewTabCommand`: Log "new-tab command received (not implemented until Phase 4)". No-op for now.
       - `ShowDesktopCommand`: Log "show-desktop command received (not implemented until Phase 2)". No-op for now.
       - Default: Log warning "Unknown IPC command"

   - Override `OnExit(ExitEventArgs e)`:
     - `LogService.Info("JoJot shutting down...");`
     - `_appShutdownCts.Cancel();`
     - `IpcService.StopServer();`
     - `DatabaseService.CloseAsync().GetAwaiter().GetResult();` (synchronous in exit path)
     - `_singleInstanceMutex?.ReleaseMutex();`
     - `_singleInstanceMutex?.Dispose();`
     - `base.OnExit(e);`

   - Add global unhandled exception handlers in OnAppStartup (before mutex):
     - `AppDomain.CurrentDomain.UnhandledException += (s, args) => { LogService.Error("Unhandled exception", args.ExceptionObject as Exception); };`
     - `DispatcherUnhandledException += (s, args) => { LogService.Error("Unhandled dispatcher exception", args.Exception); args.Handled = true; };` — per user spec: "unbreakable, degrade gracefully, never show unhandled exception dialogs"
     - `TaskScheduler.UnobservedTaskException += (s, args) => { LogService.Error("Unobserved task exception", args.Exception); args.SetObserved(); };`

4. Update JoJot/JoJot.csproj:
   - Add inside the existing PropertyGroup: `<PublishReadyToRun>true</PublishReadyToRun>` (STRT-02)
