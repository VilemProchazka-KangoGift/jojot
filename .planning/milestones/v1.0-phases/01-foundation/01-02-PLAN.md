  <files>JoJot/Services/IpcService.cs, JoJot/Services/WindowActivationHelper.cs</files>
  <action>
1. Create JoJot/Services/IpcService.cs:
   - Public static class `IpcService` in namespace `JoJot.Services`
   - Constants: `PipeName = "JoJot_IPC"`, `MutexName = "Global\\JoJot_SingleInstance"`
   - Private fields: `CancellationTokenSource? _cts`, `Action&lt;IpcMessage&gt;? _commandHandler`

   **Server side (first instance):**
   - `StartServer(Action&lt;IpcMessage&gt; commandHandler, CancellationToken appShutdownToken)`:
     - Store commandHandler
     - Create linked CancellationTokenSource from appShutdownToken
     - Launch `Task.Run(() => ListenLoopAsync(_cts.Token), _cts.Token)`
   - `ListenLoopAsync(CancellationToken ct)`:
     - While not cancelled: create `NamedPipeServerStream("JoJot_IPC", PipeDirection.In, 1, PipeTransmissionMode.Byte, PipeOptions.Asynchronous)`
     - `await server.WaitForConnectionAsync(ct)`
     - Read one line with `StreamReader`, deserialize with `JsonSerializer.Deserialize(line, IpcMessageContext.Default.IpcMessage)`
     - Invoke `_commandHandler` on the Dispatcher thread: `Application.Current.Dispatcher.InvokeAsync(() => _commandHandler(message))`
     - Catch OperationCanceledException to break cleanly, catch Exception to log and continue
   - `StopServer()`: Cancel the CTS, dispose it

   **Client side (second instance):**
   - `SendCommandAsync(IpcMessage message, int timeoutMs = 500)`:
     - Create `NamedPipeClientStream(".", "JoJot_IPC", PipeDirection.Out, PipeOptions.None)`
     - Connect with timeout: `await Task.Run(() => client.Connect(timeoutMs))` (wrapping synchronous Connect)
     - Serialize message with `JsonSerializer.Serialize(message, IpcMessageContext.Default.IpcMessage)`
     - Write line with `StreamWriter { AutoFlush = true }`
     - On TimeoutException: log warning, call `KillExistingInstances()`, do NOT exit (let caller decide)
     - On other Exception: log warning silently (per user decision: no dialog)
   - `KillExistingInstances()`:
     - `int myPid = Environment.ProcessId`
     - Loop `Process.GetProcessesByName("JoJot")`, kill any with different PID using `Kill(entireProcessTree: true)` in try/catch
     - Log each kill attempt

   **Mutex helper:**
   - `TryAcquireMutex(out Mutex mutex)`: Create `new Mutex(true, MutexName, out bool createdNew)`, return createdNew, output the mutex object for caller to hold as a field

2. Create JoJot/Services/WindowActivationHelper.cs:
   - Public static class `WindowActivationHelper` in namespace `JoJot.Services`
   - P/Invoke declarations (all `[DllImport("user32.dll")]`):
     - `SetForegroundWindow(IntPtr hWnd) -> bool`
     - `GetForegroundWindow() -> IntPtr`
     - `GetWindowThreadProcessId(IntPtr hWnd, out uint processId) -> uint`
     - `GetCurrentThreadId() -> uint`
     - `AttachThreadInput(uint idAttach, uint idAttachTo, bool fAttach) -> bool`
     - `ShowWindow(IntPtr hWnd, int nCmdShow) -> bool` (SW_RESTORE = 9)
   - `ActivateWindow(Window window)`:
     - Must be called on UI thread (assert `Dispatcher.CheckAccess()`)
     - If `WindowState == Minimized`, set to `Normal`
     - Get HWND via `new WindowInteropHelper(window).Handle`
     - Get foreground thread via `GetWindowThreadProcessId(GetForegroundWindow(), out _)`
     - Get current thread via `GetCurrentThreadId()`
     - If threads differ: `AttachThreadInput(foregroundThread, appThread, true)` -> `SetForegroundWindow(hwnd)` -> `AttachThreadInput(foregroundThread, appThread, false)`
     - Else: just `SetForegroundWindow(hwnd)`
     - Call `window.Activate()` and `window.Focus()` as final step
